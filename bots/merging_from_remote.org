* Procedure for merging a commit from a remote branch

When the user says "next commit, same procedure", follow these steps:

** Step 1 -- identify HEAD
#+begin_src shell
git log --oneline -5
#+end_src

** Step 2 -- capture commit message + stat
#+begin_src shell
git show HASH --format="%B" --no-patch
git show HASH --stat
#+end_src

** Step 3 -- read diff of non-boot files, give deep review
#+begin_src shell
git show HASH --no-color -- <non-boot files>
#+end_src

** Step 4 -- soft reset + revert totals
Soft reset discards the top commit but keeps its changes in the worktree,
allowing you to re-commit after regenerating boot artifacts. The totals
files are auto-generated and must be discarded.
#+begin_src shell
git reset --soft HEAD~1 && git checkout HEAD -- doc/totals.csv img/totals.svg
#+end_src

*** Aborting after soft reset
    If the user rejects the commit after Step 4 (before re-committing), the
    soft-reset changes are dirty in the worktree and must be discarded safely.
    NEVER use ~git reset --hard~ -- that would also destroy the previous
    commit which was processed correctly.

    Use ~git checkout HEAD~ to restore individual files:
    #+begin_src shell
    git checkout HEAD -- Makefile src/tests.til           # modified files
    git checkout HEAD -- src/test/deleted_file.sh          # deleted files
    git checkout HEAD -- doc/totals.csv img/totals.svg     # auto-generated
    #+end_src

    Verify clean with ~git status~ -- output should be empty.

    If you don't know which files are dirty, check with ~git status --short~:
    - ~M <file>~ means staged modification (restore with ~git checkout HEAD -- <file>~)
    - ~D <file>~ means staged deletion (restore with ~git checkout HEAD -- <file>~)
    - ~ M <file>~ means unstaged modification (restore with ~git checkout HEAD -- <file>~)

    Then the previous commit is safely intact at HEAD and the rejected commit
    is fully discarded.

** Step 5 -- full test suite
Run under systemd-run to limit memory (prevents OOM from memory leaks).
#+begin_src shell
systemd-run --user --scope -p MemoryMax=64G bash -c 'make clean > tmp/build.log 2>&1 && make test >> tmp/build.log 2>&1'
#+end_src

*** "Two-pass: " commits
    If the commit subject starts with "Two-pass: ", the author shipped
    boot artifacts produced by running the new compiler against itself
    (see doc/self.org). The merger MUST regenerate via ~make
    test_two_pass~ instead of ~make test~ so the committed boot/til.c
    matches the self-applied output rather than the pass-1 output.
    #+begin_src shell
    systemd-run --user --scope -p MemoryMax=64G bash -c 'make clean > tmp/build.log 2>&1 && make test_two_pass >> tmp/build.log 2>&1'
    #+end_src

*** Staged migrations (loader/parser contract changes)
    Some series can NOT be reviewed as a single rebased branch: e.g. a
    change to the import-path contract requires one commit that extends
    the loader, merged into master, BEFORE the follow-up commit that
    relies on the new form can be authored or built. If a feature
    branch ships both at once, ~git rebase~ + soft-reset + regen will
    fail at pass 1 of the second commit because til_boot was built
    from the unmerged transitional state.
    Merge each commit on its own pass through this procedure: process
    the transitional commit first (single-pass ~make test~), push to
    master, then ask the remote agent to rebase the next commit on top
    and submit it for a fresh review. See doc/self.org for the full
    rationale.

** Step 6 -- ASAN leak check (both interpret and translate paths)
Build the ASAN binary:
#+begin_src shell
make bin/til_asan > tmp/asan_build.log 2>&1
#+end_src

Check interpreted mode:
#+begin_src shell
ulimit -d unlimited && ASAN_OPTIONS=detect_leaks=1 bin/til_asan interpret src/test/strings.til > tmp/asan_interpret.log 2>&1; grep "SUMMARY" tmp/asan_interpret.log
#+end_src

Check compiled (translate) mode:
#+begin_src shell
ulimit -d unlimited && ASAN_OPTIONS=detect_leaks=1 bin/til_asan translate src/test/strings.til > tmp/asan_translate.log 2>&1; grep "SUMMARY" tmp/asan_translate.log
#+end_src

** Step 7 -- commit with exact original message + appended Co-Authored-By line
#+begin_src shell
git add -A && git commit -m "..."
#+end_src
