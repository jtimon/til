** #38 Drop til_ prefix from generated C symbols
  All ~til_~ prefix removed from builder, ext.c, dispatch.c, FFI C files.
  Generated C symbols now match the original C names (e.g. ~Token~ not
  ~til_Token~, ~malloc~ not ~til_malloc~).

  - builder.c: all ~til_~ prefix emission removed from types, functions,
    enums, structs, Vec, Array, dyn_call, dyn_has, CLI parsing, etc.
  - ext.h / ext.c: rewritten with unprefixed names
  - interpreter.c, dispatch.c: replaced ~til_~ type casts with bare types
  - ~func_to_c()~ helper maps C-colliding names: ~exit~ → ~exit_program~,
    ~sleep~ → ~sleep_ms~, ~double~ → ~double_~
  - Memory functions (malloc, calloc, realloc, memcpy, memmove): resolved
    via [[./23.org][#23]] shallow FFI — auto-FFI calls libc functions directly through
    libffi, no wrapper functions needed.
