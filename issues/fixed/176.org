# priority: high
** FIXED #176 FFI: support const char* params (avoid libc prototype conflicts)

Symptom:

   - Declaring libc functions like ~setenv~ as ~ext_func~ in Til can fail C
     compilation with "conflicting types" because generated code emits a
     prototype that does not match system headers.

   Root cause:

   - Per ~doc/qualifiers.org~, til qualifiers currently do not generate C
     ~const~ qualifiers at all. Non-~mut~ args still emit ~Type *arg~, not
     ~const Type *arg~.
   - libc headers declare many APIs with ~const char *~ (e.g. ~setenv~). If til
     codegen also emits a prototype for the same symbol but without ~const~ (or
     with a different pointer base type like ~U8 *~), the C compiler rejects it
     as a conflicting declaration.

   Current workaround:

   - Provide a thin wrapper in [[file:../../src/c/ext.c][src/c/ext.c]] (e.g. ~til_setenv~) and bind that
     from Til instead of binding libc ~setenv~ directly.

   Desired fix:

   - Add const-correct C emission:
     - When an argument is non-~mut~, emit ~const Type *~ where safe.
     - Provide an escape hatch for APIs that legitimately mutate through a
       borrowed pointer.
   - Extend FFI/type mapping to cover common C string pointer shapes:
     - ~const char *~ and ~char *~ (distinct from ~U8 *~ / ~I8 *~).
   - Reduce collisions with system headers:
     - Either avoid emitting prototypes for known libc symbols when the correct
       header is present, or ensure emitted prototypes exactly match the libc
       signature.

** Progress

   Done (commits on this branch):

   - First pass (intermediate state, since superseded -- see Final
     state below): ~is_skip_ext_decl~ in ~src/self/builder.til~
     suppressed emitted prototypes for ~setenv~ / ~getenv~ /
     ~unsetenv~ / ~putenv~ / ~system~ / ~atoi~ / ~atol~ / ~atof~ so
     ~stdlib.h~'s declaration won, and a trio of helpers
     (~is_libc_str_fn~, ~callee_param_needs_char_cast~,
     ~libc_call_return_cast~) wrapped libc call sites with explicit
     ~(char *)~ casts on ~U8~ / ~I8~ ref args and ~(U8 *)~ /
     ~(I8 *)~ casts on the call's return so the U8-vs-char and
     unsigned-vs-const conversions compiled cleanly under
     ~-Wall -Wextra -Werror~.

   - Const-correct C emission for ~ext_func~ / ~ext_proc~ prototypes:
     ~emit_param_list~ takes a ~for_ext_decl~ flag and, when true,
     prepends ~const~ to non-~mut~ Ref pointer params. Variadic /
     kwargs / shallow / own params are unchanged. ~mut~ remains the
     escape hatch. ~emit_ext_func_declarations~ also detects when an
     ~ext_func~ name is shadowed by a regular ~func~ / ~proc~
     definition (the forward-decl-via-ext_func trick used in
     ~src/self/builder.til~ for ~callee_return_ctype~ /
     ~resolve_callee_name~) and skips emitting a redundant prototype
     in that case -- the func body's own forward-decl emission is
     authoritative. For example, ~greet := ext_func(name: Str) returns
     Str {}~ in ~examples/ffi/ext_module.til~ now emits
     ~Str * greet(const Str *)~, and ~InitWindow(... title: I8)~ in
     ~src/lib/raylib.til~ emits
     ~void InitWindow(I32, I32, const I8 *)~ (= ~const char *~),
     while ~callee_return_ctype~ in ~src/self/builder.til~ keeps just
     the func-body forward decl with no const (matching the body).
   - Bullet 2 ("FFI types ~CStr~ / ~char *~ distinct from ~U8 *~ /
     ~I8 *~") is covered by til's existing ~I8~ primitive: per
     ~src/c/ext.h~, ~I8~ is a typedef for C ~char~, so ~ref I8~ params
     emit ~I8 *~ (= ~char *~) and ~const I8 *~ (= ~const char *~) with
     the const-correct emission. No new primitive type is needed.

   - Final state (post follow-ups): ~I8~ is a full first-class
     primitive (~Type.I8~, scalar+FFI plumbing through typer / initer /
     builder / dispatch), and ~Str.c_str~ itself is now ~I8 *~ (= C
     ~char *~). The raylib / tinyfd char-pointer bindings declare
     their string params as ~I8~ directly, and ~test/ext_libc.til~
     binds libc ~setenv~ / ~getenv~ / ~unsetenv~ with ~const char *~
     parameters end-to-end -- no wrappers, no per-call-site casts at
     the til level. The ~c_str_to_i8~ / ~c_str_from_i8~ helpers from
     the transitional U8/I8 bridge have been removed; ~U8~ is left
     for genuine unsigned-byte buffers (~Color~ RGBA, ~Image~ data,
     binary font output, etc.). The libc-string call-site cast
     machinery (~is_libc_str_fn~, ~callee_param_needs_char_cast~,
     ~libc_call_return_cast~) and the libc string-pointer entries
     (~setenv~ / ~getenv~ / ~unsetenv~ / ~putenv~ / ~system~ /
     ~atoi~ / ~atol~ / ~atof~) in ~is_skip_ext_decl~ have been
     retired: the generated ~const I8 *~ prototype matches
     ~stdlib.h~'s ~const char *~ declaration exactly, so til can
     emit its own prototype without conflict and call libc without a
     cast.

   - Const-correct C emission for the rest of the forward-decl paths:
     a follow-up commit pulled the top-level-ext_func filter inside
     ~emit_func_forward_decl~ (gated by a ~top_level: Bool~ argument),
     so the per-unit and per-pass loops in
     ~emit_header_defs_and_funcs_lp~ /
     ~emit_monolithic_header_func_decls_prog~ / etc. no longer leak a
     non-const prototype for symbols like ~realpath_str~ /
     ~system_cmd~ / ~stderr_print~ / ~ffi_*~ into ~til_forward.h~.
   - Hand-written FFI surface: ~src/c/ext.h~ and the matching
     ~src/c/ext.c~ definitions now use ~const~ for non-mut Ref
     pointer params on the clone / from_i64_ext / File_* / cfile_* /
     spawn_cmd / file_mtime / mkdir_p / copy_file / copy_tree /
     c_str_len / eprint_single / til_setenv / til_realpath /
     til_system / til_str_left / realpath_str / system_cmd /
     stderr_print / unlink_path / ffi_load_global_lib /
     ffi_open_user_so / ffi_user_symbol / ffi_global_symbol
     functions. Bodies got a matching ~(const char *)x->c_str~ on
     the few cast sites where it mattered. The low-level pointer
     primitives (~ptr_add~, ~to_ptr~, ~deref~, ~write_ptr~,
     ~get_payload~, ~is~, ~is_null~, ~ptr_eq~) deliberately stay
     non-const: the first five hand back / mutate interior pointers,
     and adding ~const~ to ~is_null~ / ~is~ / ~ptr_eq~ triggered
     gcc's stricter "may be used uninitialized" analysis on real
     til-emitted call sites that pattern-match
     ~buf := ... ; if (is_null(buf)) {} ; use(buf)~ -- a real-world
     non-bug.

   Regression test ~test/ext_libc.til~ binds ~setenv~, ~getenv~,
   and ~unsetenv~ directly via ~ext_func~ and asserts the round-trip
   works. 117/117 tests pass.
