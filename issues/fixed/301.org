** FIXED #301 Drop mem-total series from totals.svg, keep mem_total column in totals.csv

Created directly in fixed/ for traceability. The totals chart (img/totals.svg,
generated by examples/plot.til from doc/totals.csv via src/tests.til) plotted a
"mem-total" series whose value is the sum of the per-phase mem series
(mem_inte + mem_comp + mem_run + build_mem). It carries no information beyond
its components and, being the largest, inflated the right-axis (bytes) scale so
the drawn lines looked artificially compressed.

Fix: examples/plot.til no longer emits the mem-total polyline, dot, or legend
entry, and mem_total is excluded from mem_max / has_any_mem. The plumbing is
left intact so re-enabling is one-line per draw site:
  - Row.mem_total field, parse_csv population, the per-row clone in plot_csv
  - MEM_TOTAL_COLOR, the `series.eq("mem-total")` branches in build_points and
    series_has_data, and the `has_mem_total` flag
Are all kept, so the series returns by re-adding the polyline/dot/legend
stanzas (and re-including has_mem_total in mem_max / has_any_mem).

The mem_total column stays in doc/totals.csv (written by src/tests.til) so
the data is never lost while the visualization is off.

Regression test: src/tests.til plot test now asserts 9 polylines (was 10) and
that `mem-total` does NOT appear in the generated SVG.
