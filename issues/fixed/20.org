** #20 Dynamic arg narrowing
  Dynamic values can be passed where a concrete type is expected.
  The parameter's declared type provides context, like an explicit
  type annotation on a declaration.
  - 34.add(vec.get(0)) works (param type narrows Dynamic arg)
  - 0.add(v.get(0)).add(v.get(1)) works (typed start, chain continues)
  - bv.get(0).and(bv.get(1)) works (top-level UFCS, unique func name)
  - vec.get(0).add(35) errors with suggestion to use ref declaration
    (namespace UFCS on Dynamic is genuinely ambiguous)
  Also added arg type checking (was missing entirely):
  32.add(true) now errors instead of producing garbage.
