** #48 Builder: hoist shallow args to stack values (50x regression fix)
  The builder heap-allocates every scalar temporary, even for operations on
  shallow parameters. Vec_push does 8 mallocs per call, Str_eq does ~40
  mallocs for a 10-byte comparison, every integer literal is heap-allocated.
  This caused the 50x regression when switching from hand-written C to
  generated core.c (a9fb721).

  The ~shallow~ keyword already marks pass-by-value parameters. The builder
  should emit stack values and direct C arithmetic for shallow types instead
  of malloc/free pairs. See doc/regression.org for full analysis.
