** #86 Typer error for unused declarations (parameters + locals)

  Unused function parameters and unused local variables are now type
  errors. Prefixing the name with ~_~ suppresses the error -- the
  escape hatch the original issue specified. Unused imports are the
  remaining piece; they're split out into a new issue ([[./154.org][#154]]) because
  imports don't introduce a single named binding and til has no
  per-import suppression syntax yet.

  #+begin_src til
  x := 42          // error: unused variable 'x'
  _x := 42         // ok

  proc foo(a: I64, b: I64) {   // error: unused parameter 'b'
      println(a.to_str())
  }
  proc foo(a: I64, _b: I64) {  // ok
      println(a.to_str())
  }
  #+end_src

  Implementation in [[file:../../src/self/typer.til][src/self/typer.til]]:

  - ~TypeBinding~ gained a ~used~ bool (in [[file:../../src/self/initer.til][src/self/initer.til]]).
  - ~infer_ident_expr~ marks the resolved binding used.
  - ~infer_fcall_expr~ marks the direct-call callee binding used (so
    ~cb(a, b)~ with ~cb~ a parameter or local counts).
  - ~infer_assign_stmt~ marks the assigned binding used, so ~mut~
    out-parameters written but never read (e.g. ~parse()~'s
    ~mode_out~) aren't flagged.
  - ~check_unused_params(func_scope, expr)~ runs after each function
    body; skips ~ExtFunc~/~ExtProc~ and any param whose name starts
    with ~_~.
  - ~check_unused_locals(scope, path)~ runs after every non-type-body
    scope (function body itself, if/else/while bodies, standalone
    blocks), iterating ~scope.bindings~ and flagging any binding
    that's not a param, type def, func def, builtin, type alias,
    already used, or ~_~-prefixed. ~type_error_at(path, line, col,
    msg)~ helper added for reporting from binding metadata.
  - Autogenerated struct ~clone~ for empty structs emits ~_self~
    (otherwise every no-field struct, e.g. ~EnumDef~, would trip the
    new check).

  Cleanup of the initial ~_~-prefix pass: for truly-dead parameters
  and locals we deleted the declaration (and updated callers) rather
  than leaving the underscore. ~_~-prefix was kept only where the
  signature or the language forces the shape:

  - Lambdas bound to a ~func~/~proc~ typedef: [[file:../../src/core/dyn.til][src/core/dyn.til]]
    (~default_delete~, ~default_cmp~, ~default_inc~,
    ~default_distance~ bind to ~DeleteFn~/~CmpFn~/...),
    ~src/lib/alphabeta.til~ (~default_heuristic~,
    ~default_next_boards~).
  - UFCS namespace method receivers (the implicit ~self~ passed by
    ~x.method()~): ~unity~ in [[file:../../src/core/{u8,i16,i32,u32,u64,i64}.til][src/core/{u8,i16,i32,u32,u64,i64}.til]]
    and [[file:../../src/test/misc.til][src/test/misc.til]], ~to_str~/~eq~ in [[file:../../src/core/ast.til][src/core/ast.til]] and
    ~src/tests.til~.
  - Dispatch-table handlers matching ~DispatchFn := proc(Scope, Expr,
    mut Value) returns shallow Bool~: ~h_print_flush~, ~h_clock_ms~,
    ~h_get_thread_count~ in [[file:../../src/self/dispatch.til][src/self/dispatch.til]].
  - For-loop iteration variables and switch case payload bindings
    that don't consume their value: the language grammar requires a
    name, so ~for _i in 0..n~ and ~case Token.Name(_s)~ stay.
  - Deliberate fixtures in [[file:../../src/test/typer_errors.til][src/test/typer_errors.til]] that exist to
    trigger a different type error; renaming them would change the
    error the test exercises.

  src/test/typer_errors.til gained explicit unused-param and
  unused-local test cases. The expected type-error count in
  ~src/tests.til~ moved from 51 to 54 (52 with the new unused-param
  case, then 54 because the pre-existing block-scope test on ~inner~
  now also trips the new "unused variable" rule alongside its
  original "undefined symbol" error, plus the new unused-local
  fixture).

  Exempt from the rule (unchanged from the original issue spec):
  - Struct fields (part of a layout).
  - Enum variants (part of a type).
  - Namespace functions (may be called externally).
  - Top-level declarations (nothing calls ~check_unused_locals~ on
    the program scope).
