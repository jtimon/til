** OPEN #271 Support circular core dependencies for templated Array(Str)

   Migrating core `Array` to `Array(T)` exposed an initialization-order cycle:
   `Str.format(parts: ..Str)` needs the variadic backing array type
   `Array(Str)`, while `src/core/array.til` currently imports `str.til` for
   the legacy builtin declaration `array(type_name: Str, ...)`.

   The compiler does not currently make that cycle work. The failure presents
   as missing methods on the variadic backing array rather than as an explicit
   cycle diagnostic, e.g. `parts.len()` / `parts.get()` in `str.til` are typed
   as unresolved/bare `Array` instead of the synthesized `Array__Str`.

   *** Notes

   - Do not work around this by weakening the public `array(...)` declaration
     to `Dynamic`; fix the dependency/init model or the builtin declaration
     handling instead.
   - This blocks the clean `Array(T)` migration in #267 because `..Str` is used
     by core printing/formatting paths.
   - A useful regression is templated core `Array(Str)` being available while
     typing `src/core/str.til` variadic functions.
