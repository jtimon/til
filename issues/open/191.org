# priority: medium
** OPEN #191 Installed til computes VERSION from git at runtime

Installing the compiler system-wide works, but running the installed binary
outside the git checkout still evaluates ~get_version()~ by calling
~git rev-parse --short HEAD~ at runtime.

Repro:

  ~sudo ./bin/til install src/til.til~
  ~til repl~

Observed output begins with:

  ~til 0.0.1-fatal: not a git repository (or any of the parent directories): .git REPL -- Ctrl-D to exit~

The version string should be baked into the installed compiler at build time,
or it should fall back cleanly when no git repository is present.

Likely cause:

  ~get_version := macro() returns shallow Str { ... }~ in ~src/til.til~ is not
  being folded away during precomp/build, so the installed binary ends up
  re-running the git command at runtime instead of carrying a baked version
  string.
