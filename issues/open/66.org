** OPEN #66 Better imports: qualified namespaces

:RELATED: [[./65.org][#65]]
  Currently ~import("file.til")~ dumps all symbols into the current scope.
  Qualified imports would let you namespace them, returning a module object:

  #+begin_src til
  MyMap := import("map.til")
  MyMap.new("Str", Str.size(), "I64", I64.size())

  // vs current behavior:
  import("map.til")   // all symbols dumped into scope
  Map.new(...)        // works but pollutes namespace
  #+end_src

  This connects to [[./65.org][#65]] (standalone namespaces) — ~import()~ would return
  a namespace, and qualified access (~MyMap.new~) uses the same dot-access
  mechanism as namespace members.

  Design questions:
  - Does ~import()~ return a namespace or a module (new concept)?
  - Can you selectively import? ~from "map.til" import Map, Set~
  - Can you re-export? ~export MyMap~
  - How does this interact with the current import deduplication?
