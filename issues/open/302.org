** WIP #302 Garbager rework: make the compile-time GC the single, complete ownership authority

make test now runs the full ASAN suite and is leak-clean (e3134d5,
fe53487), but a substantial share of that greenness was bought with
compensation code OUTSIDE the garbager: manual frees and a shadow
teardown layer in interpreter.til, re-derived storage decisions and
inline fixups in builder.til, and a pile of special cases bolted onto
garbager.til itself. This issue is the deep analysis of that state and
the staged plan to fix the compile-time garbage collector system
properly, so that:

  - the garbager COMPUTES every lifetime decision once, correctly, on
    a real control-flow model;
  - the decisions are RECORDED explicitly in the typed AST;
  - builder and interpreter EXECUTE those decisions and never
    re-derive, guess, or compensate;
  - manual memory survives only where it is genuinely the allocator or
    a destructor's own body.

THE GOAL, STATED AS THE METRIC (Jorge, 2026-07-06): remove the
explicit calls to delete and free that would not be needed if the
garbager worked properly. Every stage below must name the call class
it deletes and the census section tracks the counts. Point bug-fixes
along the way are welcome but they are not the deliverable; fewer
hand-written free/delete calls is.

This is the concrete design + staging for the garbager half of #272
("single ownership authority", its suggested-staging steps 4-5), with
the inventory refreshed against current master (e333e8a). The
interpreter Value-model rewrite stays #286's plan; this issue defines
the contract that runtime must execute, and the builder-side work here
is independent of #286.

*** How we got here (the band-aid trail)

The 2026-06/07 leak-hunting arc (#109, #238, #239, #245, #250, #256,
#272, #275, #286, #299 and the "Interpreter: ..." commit series
capped by 8de7f0e) drove the suite to ASAN-clean one symptom at a
time. Nearly every fix landed at the SITE of a symptom -- an
interpreter dispatch path, a builder emission shape, one more garbager
carve-out -- because the garbager's model could not express the
lifetime rule actually being violated. doc/ownership.org's audit
sections and #275 agree on the diagnosis: the fault is architectural,
not a list of independent bugs. The fixes work, the suite is green;
the debt is that three subsystems now each own a slice of "where
values die", and every new language shape must be taught to all three
in agreement. The trail's latest entry shows the symptom in
miniature: two sessions fixed the #299 Context_delete use-after-free
in parallel with two DIFFERENT mechanisms (explicit teardown clears
in run() vs Context field order), both landed, and e333e8a then had
to remove one of them.

*** The explicit-call census (2026-07-06, vec.til read end to end)

Counts of hand-written free/delete call sites (grep of free( /
free(own / .delete( forms): src/core ~38 (numbers.til 9, tuple 6,
set 6, vec 5, ast 4, array 4, str 2, map 2), src/std ~35 (the dyn_*
containers + hashmap), src/self ~234 (interpreter.til 149,
builder.til 30, typer.til 18, loader.til 15, garbager.til 10 -- the
insertion factories themselves -- scavenger 6, constfolder 3,
eval_heap 3). Allocation typing: 56 sites bind malloc/calloc/realloc
results to `Dynamic`; 88 bind them to typed slots (own U8 fields,
`out : T = malloc(...)` locals).

Reading src/core/vec.til in full classifies every explicit call a
container makes into five classes; the same classes cover str/map/
set/array/tuple and most of the interpreter:

  a. ALLOCATION into owned storage: `data = malloc(...)` into the
     `own data : U8` field (new / with_capacity / take_prefix ctor),
     `realloc(own self.data, ...)` on grow, `out : T =
     malloc(T.size())` in pop. Stays -- this IS the allocator -- but
     see class d for the typing rule.
  b. DESTRUCTOR-TREE INTERNALS driven by the TYPE-ERASED buffer: the
     per-element `T.delete(own ptr_add(self.data, i*size), false)`
     loops (clear, move_from, unsafe_set) and the hand-written
     delete/clone overrides that exist ONLY to run those loops. The
     compiler cannot see "count elements of T behind an own U8
     pointer", so the auto-generated delete/clone cannot cover
     containers; everything else in Vec.delete (free(own self.data),
     the call_free tail) the auto-generated path would already
     handle. An owned typed buffer concept (own pointer-to-T plus a
     compiler-known count field) would let containers drop their
     hand-written delete/clone entirely. Language-design call --
     recorded here as the root cause of class b, not yet a stage.
  c. THE MANUAL MOVE IDIOM: memcpy(dst, val, size) + memset(val, 0,
     size) (push, unsafe_set, push_take, pop, clone) and its
     `other.count = 0` variant (append, move_from) -- 11 direct
     memset-defusings in core/std, mirrored by the interpreter's
     memcpy(val, val_none()) nullings. The memset exists for exactly
     one reason: to DEFUSE the delete the garbager will insert later.
     A move primitive the garbager understands deletes the whole
     idiom (stage 4, now scoped to cover core containers, not just
     the compiler-side clone-elision rewrite).
  d. MALLOC INTO Dynamic: 2 sites even inside vec.til (take_prefix's
     prefix_data, clone's new_data) among 56 repo-wide. Design ruling
     2026-07-06: Dynamic is for SKIPPING TYPE CHECKING (dyn features,
     type-erased borrows like to_ptr/ptr_add results), NOT for owning
     storage; owned raw buffers are typed (own U8 fields, typed
     locals like pop's `out : T`). These 56 sites migrate to typed
     bindings and binding an allocator result to Dynamic becomes a
     type error. See the reworked rule in New diagnostics.
  e. DESTRUCTOR-TAIL PROTOCOL: `if call_free { free(own self) }`
     repeated verbatim in every hand-written delete (all 9 of
     numbers.til's calls are this and nothing else). Protocol
     boilerplate that belongs to the delete contract / codegen, not
     to every author.

What this census says about the goal: the interpreter's 149 calls are
the #286-gated shadow layer (already inventoried below); of the
REST, class c (the move idiom) and class d (Dynamic ownership) are
garbager/typer work with no interpreter dependency, class b needs the
typed-buffer language decision, class a stays, class e dies with the
recorded-decision destructor contract (stage 4/6).

*** The garbager's output today (the undocumented contract)

What the pass actually produces, and who consumes it:

  1. Inserted destructor statements: `Type.delete(x, cf)` /
     `free(own x)` FCall nodes (make_delete_call / make_free_call in
     typer.til) at last use, before reassignment, and on exit edges.
  2. Deletes attached as EXTRA CHILDREN of Return nodes ("evaluate
     children[0], then run children[1..], then return").
  3. save_old_delete flags on Assign/FieldAssign -- set from TWO
     places (typer's infer_assign_stmt and garbager's
     insert_assign_delete); the builder emits the old-value delete
     inline, in stack and heap variants.
  4. own_type promotion on Decl (promote_own_transferred_locals):
     owned aggregate locals flip Shallow -> Own so codegen heap-boxes
     them.
  5. Clone-elision rewrites: at a dying local's last use,
     `lhs = local.clone()` / `holder.f = T.clone(local)` is rewritten
     back to bare `local` so codegen emits a move -- ownership
     transfer expressed by silently EDITING an expression.
  6. Wrapper frees after a Shallow-field value-copy transfer
     (transfer_is_shallow_field_value_copy + make_free_call): the one
     "half a transfer" shape (doc/ownership.org).

Consumers: the builder re-derives about half of these decisions from
scratch (D3/D4 below) and the interpreter bypasses or re-implements
the other half (inventory below). Nothing checks that the three
agree; the test corpus under ASAN is the only referee.

*** Structural deficits (the root causes)

**** D1. Statement-index liveness; no CFG

LocalInfo is three I32 indexes into ONE body's children Vec:
decl_index, last_use, own_transfer. Paths do not exist in this model.
Every path-shaped bug became a bolted-on special case:

  - sink_conditional_transfer_deletes + sink_nontransfer_paths:
    transfer inside one if-branch, delete grafted into the others.
    (CORRECTION 2026-07-05: an earlier revision here repeated
    doc/ownership.org's stale claim that Str was excluded from
    sinking; Str is path-handled. Params WERE excluded wholesale --
    verified leaks, fixed with LocalInfo.dtor_transfer, see Progress);
  - the own_transfer <= stmt_idx pre-transfer rules duplicated in
    insert_exit_deletes_into_stmt and insert_nested_exit_deletes: an
    exit edge before the transfer stmt still owns the local;
  - return_only threading through insert_exit_deletes so break/
    continue edges are cleaned in loops but not across them.

A single own_transfer index cannot represent two conditional
transfers, a transfer inside a loop, or per-branch last uses; the
current answers are "first transfer wins", "extend to scope end", or
"skip". check_use_after_own_transfer compares indexes, so a use after
transfer on a DIFFERENT path is a false error shape waiting to
happen, and a use-before-transfer ordering bug on one path is
invisible.

**** D2. Name-string dataflow; no alias model

Everything is expr_uses_var(stmt, name) textual matching. View/alias
tracking is the collect_hoist_taint machinery (#238/#250): a binding
participates iff its name starts_with("hoisted__"), so user-written
views get no tracking at all, and compiler temps get transitive-taint
OVER-extension (acknowledged in extend_hoist_view_lifetimes' header).
alias_used_in_stmts re-walks Decl statements to catch `ref x := local`
before an exit delete. extend_ref_local_lifetimes extends a
FieldAssign borrow source to END OF BODY because "without aliasing
analysis we can't pin a tighter bound" (its own comment).

**** D3. The stack-vs-heap decision is made twice

The garbager guesses is_heap per binding from RHS shapes (LiteralStr,
fcall_returns_shallow, is_pod_enum_clone_wrap, funcptr captures,
BodyValue) in collect_scope_locals + promote_own_transferred_locals.
builder.til INDEPENDENTLY re-derives the same classification into
ctx.builder_func.stack_locals / stack_local_types in emit_stmt_decl,
with its own copies of the literal / shallow-return / POD-enum-clone /
clone-of-stack-local shape checks (and its own names for the shared
predicates: is_pod_enum_clone_call vs is_pod_enum_clone_wrap,
fcall_is_shallow_return vs fcall_returns_shallow). Every mismatch is
UB (free of a stack pointer) or a leak, and every new RHS shape must
be taught to both sides. The builder then papers over residual
disagreements at every consumption site:

  - emit_as_ptr's `_oa` malloc-copy wrap (own arg fed from a stack
    local);
  - is_noop_delete_stmt (skip garbager deletes that are no-ops on
    stack scalars / POD enums);
  - the delete-arg rewrite to `&local` + call_free=false for stack
    locals, re-deriving the call_free literal via delete_call_free_lit;
  - return boxing of stack locals (`*_r = local`) plus cap laundering
    `if (_r->cap == TIL_CAP_LIT) { _r->cap = TIL_CAP_VIEW; }`.

**** D4. Decisions travel through fragile channels

  - save_old_delete set by both typer and garbager (output 3 above);
  - move-instead-of-clone expressed by rewriting `x.clone()` to bare
    `x` (output 5), which downstream phases must then re-recognize as
    a move (the #256 own-field vs value-field wrapper-free split; the
    interpreter's scope_set_owned source-nulling);
  - exit deletes ride as Return children, so every one of the ~8
    return-emission shapes in builder.til must unpack "children[1..]
    are cleanup", and body_diverges must know the garbager appends
    cleanup after terminators;
  - the Shallow-field wrapper free depends on re-detecting the ctor
    shape at delete-insertion time (positional/named-arg field
    resolution in transfer_is_shallow_field_value_copy);
  - transfer detection (expr_transfers_own / fcall_has_own_arg) only
    matches bare Ident args; ctor own-field detection resolves fields
    positionally and via NamedArg by name -- each a small parser of
    shapes the typer has already resolved once.

**** D5. Owned temporaries exist only by hoisting convention

The garbager sees named locals in scope.bindings; every owned
intermediate value must have been lifted into a `hoisted__*` Decl by
the typer's hoist passes to be visible. The contract is implicit: any
expression shape the hoister misses is INVISIBLE to the ASAP pass and
leaks silently (the f().field class recorded in #272), and the
builder even hand-frees one shape itself (the "Asap fix" in the
Return emission: a shallow return of a hoisted temp does DEREF +
free() inline because insert_exit_deletes_into_stmt skips locals used
in the return expr).

**** D6. Global flags for shape carve-outs

ctx.typer_in_main_func (the builder heap-allocates main's CLI params,
so the garbager must delete Ref params there and only there),
ctx.typer_in_shallow_return (keep the returned local Shallow so the
value-copy return works), is_program_scope + the hoisted__ prefix
carve-out at program scope. Each is global state threaded through
Context to flip individual predicates deep inside the pass.

*** Target design

One pass, one model, one recorded output.

**** Ownership dataflow on a real CFG

Build a per-body CFG (if/else, while with break/continue back and
exit edges, switch cases, early returns, catch). For each owned
binding run a forward dataflow with this per-point state:

    Uninit | Owned | Moved(site) | Borrowed-from(owner set)

Transfers (own arg, own-field assign, own capture, value-copy into a
Shallow field), borrows (ref decls, ref-field assigns, view-returning
calls), uses, and reassignments are edges discovered on the typed AST
by binding identity -- not name strings, not the hoisted__ prefix.
Destructor insertion = at every CFG edge where a binding drops from
Owned to dead (last use, scope exit, early exit, reassign), per path.
That subsumes, with no special cases:

  - sink_conditional_transfer_deletes: still-Owned on the
    non-transfer path -> delete inserted on that path;
  - the pre-transfer early-exit rules: an exit edge upstream of the
    transfer node still sees Owned;
  - check_use_after_own_transfer: use while Moved = error, now
    path-sensitive instead of last_use > own_transfer;
  - extend_ref_local_lifetimes / extend_hoist_view_lifetimes /
    collect_hoist_taint / alias_used_in_stmts: borrow edges keep the
    owner alive exactly as long as any borrower -- user-named views
    included, over-taint gone;
  - double transfer on one path = a new, currently-undetectable
    error, reported at the second site.

**** New diagnostics: forbid the obviously-leaking shapes

The same per-path ownership state that places deletes also REPORTS the
shapes that today leak or double-free silently:

  - Use-after-move / double-move, path-sensitive (above). Includes the
    transfer-inside-a-loop of a local declared outside the loop: the
    single own_transfer index cannot represent the second iteration, so
    today this compiles silently into double-free C.
  - Dynamic never OWNS (REWORKED 2026-07-06, superseding the 07-05
    "leaked Dynamic" enforcement rule). Design ruling: Dynamic exists
    to skip type checking -- dyn features and type-erased BORROWS
    (to_ptr / ptr_add / deref results) -- not to hold owned storage;
    owned raw buffers are TYPED (own U8 fields as in Vec.data, typed
    locals as in Vec.pop's `out : T = malloc(T.size())`). Two rules
    replace the old plan of enforcing the manual-free contract:
      1. Binding an allocator result (malloc / calloc / realloc /
         EvalHeap.heap_alloc) or any owned pointer to a `Dynamic`
         binding/field/param is a TYPE ERROR. The 56 existing
         `: Dynamic = malloc(...)` sites migrate to typed bindings
         first (vec.til's take_prefix/clone among them).
      2. Allocator-rooted TYPED locals become ordinary garbager-owned
         values: freed automatically at last use / exit edges like
         any owned local, transferable via own args/fields. The
         qualifiers.org manual contract ("must be explicitly freed
         with free(own buf)") is thereby REMOVED, not enforced --
         hand-written free(own buf) on locals becomes redundant and
         then rejected, deleting that call class instead of policing
         it (this is the goal metric applied to class d of the
         census). The 07-05 codegen fix (Progress item 4) already
         unblocked pointer-value emission everywhere.
  - Unhoisted owned temp (stage 5 below).
  - No byte-copy of managed aggregates (with stage 4; #275 step 3):
    extend check_move_managed_dest with its dual.

Verified 2026-07-05 (tmp experiment), FIXED same day (Progress item
4): `buf : Dynamic = malloc(8)` + `free(own buf)` at PROGRAM scope
emitted `free(({ void *_oa = malloc(sizeof(void)); *_oa = buf; _oa;
}));` -- two -Werror failures (void deref / void expression).
emit_as_ptr treated the program-scope Dynamic local as a value global
and applied the own-box malloc-copy wrap; an own Dynamic arg must be
the pointer itself. The same free inside a proc body always compiled
fine (Dynamic locals never enter stack_locals).

**** Explicit, recorded decisions (backends stop deciding)

  - Declaration gets a storage decision set ONCE by the garbager:
    heap-box vs stack-value (replaces both the garbager is_heap guess
    and builder's stack_locals re-derivation; emit_stmt_decl consumes
    it).
  - Destructor statements carry their full contract when inserted:
    receiver form (value vs pointer), call_free, arg_is_own. The
    builder stops re-deriving call_free/receiver from stack_locals,
    and no-op deletes are simply NOT INSERTED (is_noop_delete_stmt
    dies).
  - Moves become explicit: an Assign/FieldAssign/arg consuming a
    dying local is MARKED as a move (a flag beside save_old_delete /
    the own-arg bit) instead of the clone-call textual elision;
    builder and interpreter implement "move" once each.
  - save_old_delete is set in exactly one place (the garbager);
    typer's three sites move in.
  - Return nodes stop carrying delete children: the garbager hoists
    the return operand into a temp Decl (returns become
    `return <ident>`) and inserts exit deletes as ORDINARY statements
    before the Return. The ~8 special return emissions in builder.til
    collapse into the plain form.

**** Enforced temp contract

Keep the hoisting division of labor (the pass sees named locals), but
make the contract CHECKED: after typing, an owned-aggregate-producing
expression in a non-Decl position that no transfer consumes is an ICE
("unhoisted owned temp at <loc>"), not a silent leak. That converts
D5's invisible class into loud build errors and produces the real
residual-shape list. If that list turns out long, the follow-up is to
let the ownership pass insert temps itself; start with the check.

**** Special cases normalized away

  - main() CLI params: mark them own_type = Own at init time -- main
    is never called from til code and the builder's argv prologue IS
    their constructor. ctx.typer_in_main_func dies.
  - shallow-return locals: "local whose only escape is the value-copy
    at a shallow return" becomes an ordinary storage classification
    on the CFG. ctx.typer_in_shallow_return dies.
  - Str keeps its runtime cap tag (CAP_LIT / CAP_VIEW / owned), and
    Str.delete remains the single place that reads it. Codegen stops
    emitting its own cap tests and laundering; where the garbager
    knows statically (literal RHS) it emits the right delete form,
    where it cannot, the delete call is emitted and Str.delete
    decides at runtime.

*** Staged plan

Each stage lands suite-green. Garbager-semantics stages are
`Two-pass:` commits (the pass changes what C is emitted for the
compiler's own sources; see doc/self.org "Two-pass build" and
doc/ownership.org "How to verify"). Verification per stage is
`make clean && make test` or `make clean && make test_two_pass`
accordingly.

Stage 0. Contract snapshot + audit tooling. [plain commit]
  Write doc/garbager.org documenting today's exact outputs (the six
  above) and the target contract. Add a debug dump (e.g. TIL_GC_AUDIT
  env var) that prints, per body: bindings, storage decision,
  insertion sites, transfers, borrows. Golden a handful of bodies in
  a new test so stages 1-5 show their decision diffs in review
  instead of only via ASAN.

Stage 1. Single-source storage classification. [Two-pass]
  The garbager computes the storage decision per binding (today's
  is_heap logic, centralized) and RECORDS it on Declaration. The
  builder consumes the recorded decision in emit_stmt_decl; during
  transition it ASSERTS agreement with its old shape checks (loud
  panic on mismatch), then the duplicated checks are deleted --
  including the is_pod_enum_clone_call / is_pod_enum_clone_wrap and
  fcall_is_shallow_return / fcall_returns_shallow twins. emit_as_ptr's
  `_oa` box and is_noop_delete_stmt become dead or assert-only.

Stage 2. Return normalization. [Two-pass]
  Stop attaching deletes as Return children: hoist return operands to
  temps, insert exit deletes as preceding statements. Builder return
  emission collapses; the body_diverges caveat goes away. Also the
  precondition for Stage 3 (Return becomes a CFG leaf, not a cleanup
  carrier).

Stage 3. The CFG ownership pass. [Two-pass, the big one]
  Implement the dataflow above inside garbager.til, replacing
  LocalInfo's three indexes. Keep insertion output identical on
  straight-line bodies (checked via Stage 0 goldens); branchy bodies
  become path-correct. Delete in the same or immediate follow-up
  commit: sink_conditional_transfer_deletes + sink_nontransfer_paths,
  the pre-transfer exit rules, index-based
  check_use_after_own_transfer (becomes a dataflow diagnostic),
  extend_ref_local_lifetimes, extend_hoist_view_lifetimes,
  collect_hoist_taint, rhs_depends_on_var, var_aliases_target,
  alias_used_in_stmts. Borrow edges come from binding identity plus
  view-returning-call classification (returns ref / substr class),
  not the hoisted__ prefix. Add the path regression tests (below).
  The "New diagnostics" set rides this same dataflow: the
  path-sensitive use-after-move family and the Dynamic-never-owns
  pair land here (allocator-into-Dynamic type error + auto-freed
  allocator-rooted typed locals). Calls removed: every
  `free(own buf)` on a local raw buffer (census class d), after the
  56 `: Dynamic = malloc(...)` sites migrate to typed bindings.

Stage 4. Explicit moves + unified reassign deletes. [Two-pass]
  Move markers replace the clone-elision rewrites; save_old_delete
  decisions consolidate into the garbager; the Shallow-field
  value-copy wrapper free becomes a decision recorded at the transfer
  site instead of re-detected. Builder and interpreter implement the
  move marker once (pointer move / scope_set_owned nulling). This is
  also where #275's step-3 typer enforcement (no memcpy of managed
  aggregate bytes) can land, since moves are now first-class.
  SCOPE WIDENED 2026-07-06: the move must be a USER-FACING primitive,
  not only the compiler-internal marker -- core containers implement
  moves today as memcpy(dst, val, size) + memset(val, 0, size) (or
  `other.count = 0`), where the memset exists solely to defuse the
  garbager's later delete (census class c: push, unsafe_set,
  push_take, pop, clone, append, move_from in vec.til alone, ~11
  memset-defusings across core/std, plus the interpreter's
  val_none() nullings). Calls removed: the whole idiom -- and with
  it #275's byte-copy enforcement becomes possible, since the
  legitimate byte-moves now have a first-class spelling. The
  destructor-tail boilerplate (`if call_free { free(own self) }`,
  census class e) folds into the recorded destructor contract here
  or in stage 6.

Stage 5. Temp-contract enforcement. [Two-pass]
  The unhoisted-owned-temp ICE. Fix what it flushes out (the
  f().field class) by extending hoist coverage or pass-owned temp
  insertion. (The program-scope `free(own p)` invalid-C fix already
  landed 2026-07-05, Progress item 4.)

Stage 6. Builder compensation removal. [Two-pass]
  With 1-5 in, delete the remaining builder fixups: swap_freed
  accounting, CAP laundering at return boxing, the FieldAssign cap
  test, the "Asap fix" inline free. Any removal that breaks a test
  exposes a REAL garbager gap: fix the gap in the pass, then delete
  the fixup. Never the reverse order.

Stage 7. Interpreter compensation removal -- rides #286.
  Gated on #286's raw-bytes runtime (delete bodies must be executable
  against real layouts before the interception layer can die). This
  issue's stages make the AST contract complete and explicit so
  #286's stages consume it; once #286 reaches its stage 6, the shadow
  teardown/clone tree, the del_method interception,
  free_temp_arg_value / free_non_own_temp_args, deep scope_free,
  guard_own_param_skip and the borrowed forgeries reduce to #275's
  two rules: interpret the garbager's delete/free/move, and free the
  interpreter's own bookkeeping.

*** Verification

  - Every stage: `make clean && make test` (the full ASAN suite is
    the standing gate); Two-pass stages: `make clean &&
    make test_two_pass`.
  - Leak probes stay at zero true leaks:
      TIL_LEAK_PROBE=1 ASAN_OPTIONS=detect_leaks=1 bin/til_asan interpret test/strings.til
      TIL_LEAK_PROBE=1 ASAN_OPTIONS=detect_leaks=1 bin/til_asan translate test/strings.til
  - New regression tests for the path cases the index model cannot
    express (today each is either a special case or silently
    conservative): transfer in one branch (then/else orders, nested
    else-if), transfer inside a loop body, early return before AND
    after a transfer, break/continue with live owned locals, a
    user-named (non-hoisted) view outliving its statement, view of a
    view, two locals conditionally transferred into the same own
    field, use-after-move on one path only (must error), self-assign
    `x = x.clone()` (must keep the clone), transfer inside a loop of an
    outer local (must error), and the Dynamic-never-owns pair:
    `: Dynamic = malloc(...)` (must error), an allocator-rooted TYPED
    buffer local auto-freed on every path with no hand-written free
    (ASAN-verified), the same local transferred via an own arg/field
    (no double free), a to_ptr/ptr_add Dynamic borrow never flagged
    and never freed, and pop-style `out : T = malloc(...)` returned
    to the caller (caller's asap pass frees it).
  - Stage 0's GC-audit goldens diff the pass's decisions directly, so
    behavior changes are visible in review rather than only as ASAN
    outcomes.

*** Non-goals

  - No runtime GC, no refcounting: the ASAP single-owner model stays
    (doc/ownership.org, "Relation to Mojo").
  - No qualifier surface changes: own/ref/shallow/mut semantics stay
    as documented in doc/qualifiers.org.
  - Not the interpreter Value/heap rewrite -- that is #286; this
    issue fixes the contract it executes.
  - Not the scavenger / EvalHeap designs.
