# priority: medium
** WIP #205 Collapse Primitive enum + per-primitive Type variants into Type.Struct(name)

   Issue #203 grouped primitives behind a single ~Type.Primitive~
   variant via a side enum. #205 is the next simplification:
   drop that grouping AND the still-Type-local primitive variants
   (~Type.I64~, ~Type.U8~, ~Type.I8~, ~Type.I32~, ~Type.U32~,
   ~Type.U64~, ~Type.F32~, ~Type.Bool~) in favour of treating
   every primitive as ~Type.Struct(name)~ -- the same shape ~Str~,
   ~Vec~, ~Map~ etc. already use.

   Goal end state:

   #+begin_src til
   Type := enum {
       Unknown, None,
       Struct: Str, StructDef,
       Enum: Str, EnumDef,
       FuncDef, FuncPtr, Dynamic,
       Custom: Str,
   }

   // No Primitive enum, no Type.Primitive variant.
   // I64 / U8 / I8 / I16 / I32 / U32 / U64 / F32 / Bool all bind
   // and round-trip as Type.Struct("<name>".clone()).

   is_primitive := func(t: Type) returns shallow Bool {
       switch t {
       case Type.Struct(name):
           if name.eq("I64")  { return true }
           if name.eq("U8")   { return true }
           if name.eq("I8")   { return true }
           if name.eq("I16")  { return true }
           if name.eq("I32")  { return true }
           if name.eq("U32")  { return true }
           if name.eq("U64")  { return true }
           if name.eq("F32")  { return true }
           if name.eq("Bool") { return true }
           return false
       case: return false
       }
   }
   #+end_src

   Net savings: drops the ~Primitive~ enum, the ~Type.Primitive~
   variant, eight per-primitive ~Type.X~ variants, the typer's
   ~make_til_type_expr~ Primitive special case, the builder's
   ~emit_dyn_type_to_str_body~ Primitive arm, the interpreter's
   ~h_dyn_type_to_str~ Primitive arm, dozens of per-variant
   ~case Type.X:~ switch arms across ast/typer/builder/loader,
   and every ~Type.eq~ per-variant tag-equality line.
   Construction sites collapse from ~Type.I64~ or
   ~Type.Primitive(Primitive.I16)~ to a single
   ~Type.Struct(name.clone())~ shape.

   *** Plan

   - [X] Step 1 (this issue's first commit): introduce the
     ~is_primitive~ helper. Wire it up at the three OR-chain
     sites that already exist (~obj_method_type_name~,
     ~is_scalar~ in shallow-param validation, ~emit_til_default~).
     No representation change yet; the helper's body still ORs
     the current ~Type.X~ + ~Type.Primitive~ variants. Lands as
     a tiny, safe commit on top of master so callers and
     reviewers can see the new shape and stop replicating the
     OR pattern on new code.

   - [ ] Step 2: migrate the still-Type-local primitives
     (~I64~, ~U8~, ~I8~, ~I32~, ~U32~, ~U64~, ~F32~, ~Bool~) to
     ~Type.Struct(name)~. Atomic commit:
     - Update ~type_from_name~ in ~typer.til~ + ~initer.til~ to
       return ~Type.Struct("<name>".clone())~ for these names.
     - Update ~register_builtin_types~ initer/typer paths so
       ~builtin_type~ for these names is ~Type.Struct(...)~
       instead of the dedicated Type variant.
     - Sweep every ~Type.I64~ / ~Type.U8~ / ~Type.I8~ /
       ~Type.I32~ / ~Type.U32~ / ~Type.U64~ / ~Type.F32~ /
       ~Type.Bool~ in compiler source (construction, ~is(...)~,
       ~case Type.X:~ switch arms) and rewrite to the
       ~Type.Struct(name)~ shape -- either by-name dispatch
       under a single ~case Type.Struct(s):~ or by calling
       helpers (~is_primitive~, ~til_type_name~,
       ~til_type_name_c~).
     - Drop the now-dead variants from ~Type~.
     - Update ~is_primitive~'s body to read these primitives
       from the new ~Type.Struct(name)~ shape.

   - [ ] Step 3: migrate ~I16~ (currently in ~Type.Primitive~)
     to ~Type.Struct("I16")~. Same mechanics as Step 2.
     - Drop the ~Type.Primitive~ variant.
     - Drop the ~Primitive~ enum entirely.
     - Drop the ~make_til_type_expr~ Primitive case, the
       ~emit_dyn_type_to_str_body~ Primitive arm, the
       ~h_dyn_type_to_str~ Primitive arm.
     - Final ~is_primitive~ body matches the goal sketch above.

   Step 2 vs Step 3 order is interchangeable. The atomic-commit
   constraint comes from the typer's checks/construction sites
   being tightly coupled to the binding's runtime representation:
   changing only the bindings (or only the checks) desyncs them
   and breaks ~make test~. So each step ships the binding
   update, the construction-site update, the check/switch
   rewrite, and the variant removal together.

   *** Why medium priority

   Pure simplification -- correctness is the same. Useful for
   future ~auto-shallow~ / ~is_numeric~ refactors that benefit
   from a single canonical ~is_primitive~ test, and to retire
   the per-variant ~Type~ tags that today have to be re-stated
   at every switch and every OR chain. No external behaviour
   change.
