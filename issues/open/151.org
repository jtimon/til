** WIP #151 32-bit targets: wasm32 and templeos-x86

:DEPENDENCIES: [[./81.org][#81]] [[./232.org][#232]]

  [[file:../../src/self/targets.til][src/self/targets.til]] already has ~Target.Wasm32~ and
  ~Target.TempleosX86~ in the enum (mirrored from rstil), with
  toolchain selection (~clang --target=wasm32~, ~holyc~) and
  executable extension (~.wasm~, no ext) wired up. Nothing actually
  builds for these targets because til's generated C assumes 64-bit
  pointers everywhere (~USize~, raw pointer arithmetic), and the
  32-bit targets would generate C that's unsound on 32-bit ABIs.

  Progress so far (via [[./232.org][#232]] LinuxX86 canary):

  * Fixed ~get_payload~ in [[file:../../src/c/ext.c][src/c/ext.c]] to use ~sizeof(void*)~
    instead of ~sizeof(I64)~ for the tag-offset computation. On 64-bit
    both are 8, but on 32-bit ~sizeof(I64)=8~ (off by 4) while
    ~sizeof(void*)=4~ (correct). This was the primary crash in all
    32-bit til programs (Type_delete reading garbage payload pointers).

  * Fixed ~dyn_size_of~ in [[file:../../src/self/builder.til][src/self/builder.til]] to use
    ~sizeof(Dynamic)~ instead of hardcoded 8.

  * Cross-compiled ~examples/hello_script.til~ (~echo Hello World~) and
    ~tmp/hello_simple.til~ (~println(~["Hello"]~)~) for linux-x86
    with ~gcc -m32~ and verified they run correctly under ~qemu-i386~.

  Remaining (per [[./232.org][#232]]):
  - Build 32-bit vendor libs (libffi, raylib, tinyfd)
  - Add Makefile targets and CI job
  - Run full test suite for linux-x86 via qemu
