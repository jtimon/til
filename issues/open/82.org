# priority: low
** OPEN #82 POSIX ~--~ end-of-options in til CLI argument parsing

Decide whether til's CLI argument parser should support ~--~ as
  "end of options" (POSIX getopt convention). Most Unix tools support
  it. Currently til.til parses ~-o~ and ~-c~ flags before the source
  path — no ~--~ handling.

  Probably not needed until the CLI gets complex enough flags that
  ambiguity becomes a real risk. Low priority.
