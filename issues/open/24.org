** OPEN #24 Music theory examples: ongoing improvements

Permanent umbrella issue for GUI music theory learning examples.

Current examples:

- `examples/7modes.til`: renamed from `examples/music.til` after the
  shared helper module became `lib/music.til`. Shows note/mode controls,
  a pentagram, scale playback, note click playback, triad labels, PT/ES/EN
  mode-name labels, and a compact playable piano.
- `examples/piano.til`: standalone 3-octave QWERTY piano with scrolling
  piano-roll visualization.

Recent progress:

- Extracted common piano/music data into `lib/music.til`: QWERTY keycodes,
  key labels, C3-B5 piano frequencies, natural/black-key layout tables,
  solfege names, and letter pitch names.
- Updated `examples/piano.til` and `examples/7modes.til` to share the same
  key/frequency mapping, so a keyboard key means the same pitch in both.
- Added QWERTY labels to the compact piano in `examples/7modes.til`.
- Added keyboard and mouse/touch input for the compact piano in
  `examples/7modes.til`.
- Made pentagram notes and piano keys highlight each other in both
  directions when the played key is part of the displayed scale.
- Removed the stale solfege/letter notation radio button. Individual note
  labels now use solfege (`Do`, `Re`, `Mi`), while triad labels use
  letter-based chord symbols (`C`, `Dm`, `Bdim`).
- Kept the piano-roll labels in `examples/piano.til` as letter pitch names,
  because piano-roll rows represent absolute pitches, not chord symbols.

Notes:

- The old mode-button rendering note from [[./44.org][#44]] is stale; manual
  testing of the current `examples/7modes.til` showed the mode controls and
  removed radio row behaving correctly.
- `examples/7modes.til` still calculates scale spelling separately from
  pitch frequency so staff position, accidentals, and audio stay aligned.
