# priority: low
** OPEN #24 Music scale viewer: ongoing improvements

GUI music theory learning tool (examples/music.til). Uses mode gui
  (raylib). Click note + mode buttons to see/hear scales on pentagram.
  Click individual notes on staff to play them.

  Known bug: mode buttons not rendering after from_i64 cleanup ([[./44.org][#44]]).
  Pre-existing issue — was crashing before, now runs but mode buttons
  are missing. Needs investigation.

  Features: 7x3 note buttons (with b/#), 7 Greek mode buttons,
  graphical pentagram with notes, inline triad labels (Do-M, Re-m,
  Mib-dim), reference labels on both sides of staff, click-to-play
  individual notes, audio playback via play_tone (til-native sine
  wave), language radio (PT/ES/EN), notation radio (solfege/american,
  translated per language), resolution-independent layout with fixed
  staff positioning (precalculated from note range), 2-octave piano
  roll visualization with key highlighting, DejaVuSans custom font
  (bundled in assets/), Makita teal color scheme for UI elements.

  Code is organized with extracted helper functions: triad_type,
  note_name, acc_text, staff_y, lang_str, start_scale, play_note,
  draw_note, recompute_scale, draw_piano. Data-driven: note names,
  mode names, language strings, notation labels, triad suffixes stored
  in constant arrays (LANGS array-of-arrays pattern). Uses for..in
  range loops throughout.

  Octave calculation uses separate diatonic octave (for staff position)
  and chromatic octave (for frequency), correctly handling scales that
  cross the C boundary (e.g. F Locrian where Cb is B4 not B5).
