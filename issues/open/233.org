*   Current state:
    - /// doc comments support org-style metadata lines: ~#+doc: false~, ~#+group:~, ~#+since:~, and ~#+deprecated:~.

    - Doctests verified by ~make test~.

*   Architectural Learnings (from Python/Sphinx and Elixir/ExDoc):
    - *Cross-Referencing*: Deterministic object IDs based on qualified names (for example, ~domain:module.class.func~) eliminate the need for manual link mapping. Sphinx domains and ExDoc nodes heavily utilize this.
    - *Docstring Parsing*: Docstrings are treated as markup and parsed dynamically. ExDoc compiles an intermediate HTML AST (~ExDoc.DocAST~) in memory before writing to disk, allowing metadata injection (anchors, cross-refs).
    - *Build Integration*: Python/Sphinx imports modules at build time (~autodoc~), uses ~inspect~ to read ~__doc__~, and parses as reST. Elixir/ExDoc embeds docs in compiled ~.beam~ files as EEP-48 ~Docs~ chunks; at runtime, ~Code.fetch_docs/1~ loads them.
    - *Output Formatting*: Separation of concerns. Sphinx uses Jinja2 templates; ExDoc uses EEx templates. Both walk a format-agnostic internal AST.
    - *Search/Navigation*: ExDoc dumps an inverted index of all documentation into a JSON blob embedded in Javascript for instant client-side search. Navigation sidebars are data-driven from module groupings.
    - *Doctests*: Elixir dynamically extracts docs via ~Code.fetch_docs/1~, parses lines starting with ~iex>~, executes them, and asserts the output matches the following lines. Handled by the test runner (~ExUnit.DocTest~), not the doc generator itself.
