# priority: high
** OPEN #181 Declared-type inference for literal expressions

When a variable is declared with an explicit type annotation, the
   initialiser expression should be coerced to match:

   #+begin_src til
   half : F32 = 1 / 2      // should infer F32 context, produce 0.5
   #+end_src

   Currently ~1 / 2~ stays ~I64~ (integer division -> 0), and the
   typer rejects it with "declared as F32 but value is I64".  The
   typer should see the declared type ~F32~ and desugar
   ~1 / 2~ to ~1.to_f32() / 2~ automatically (or equivalent), without
   adding an extra compiler pass (see [[./158.org][#158]], one-pass-per-phase goal).

   Regression test to add to [[file:../../src/test/regression.til][src/test/regression.til]] once fixed:

   #+begin_src til
   test_bug181_declared_type_inference := test() {
       half : F32 = 1 / 2
       assert(__LOC__, half.eq(1.to_f32() / 2.to_f32()))
       inch : F32 = 1 / 10
       assert(__LOC__, inch.eq(1.to_f32() / 10.to_f32()))
   }
   #+end_src
