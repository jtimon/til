** OPEN #70 Precomp: constant folding for all types (not just scalars)

The builder emits top-level constants as heap-allocated runtime
  variables. Precomp currently folds scalar expressions (I64, Str,
  Bool) but not struct-valued or collection-valued ones.

  Broader scope: precomp should fold ANY compile-time-known expression
  into a constant value in the generated code. Examples:
  - ~CAP_LIT := max_u64()~ → ~#define CAP_LIT ULLONG_MAX~
  - ~MODE_SCRIPT := Mode(name = "script")~ → ~Mode MODE_SCRIPT = {...}~
  - ~core_modes := {"script": Mode(...), ...}~ → static Map
  - Fibonacci via macro returning Vec → static array

  The syntax already works: ~ref MODE_SCRIPT : Mode = core_modes.get("script")~
  is a typed reference to a compile-time-known value. Precomp just needs
  to evaluate it and the builder needs to emit the result as a constant.

  CAP_LIT/CAP_VIEW no longer rely on [[file:../../src/c/pre70.h][src/c/pre70.h]]. Handwritten C now
  gets them through the normal generated headers, while the remaining
  broader constant-folding work stays here.

  Current status after the real ~[[./104.org][#104]]~ fix:
  - ordinary top-level globals are now usable from handwritten C through
    generated ~extern~ declarations in ~boot/til_forward.h~
  - but namespace/static-field globals still do not have the same final
    generated-C story
  - so the remaining work to make things like ~Str.CAP_LIT~ / generated
    ~Str_CAP_LIT~ usable through the same final generated-C path still
    belongs here under ~[[./70.org][#70]]~, not under ~[[./104.org][#104]]~

  Also blocks ~const~ in hand-written C: the builder doesn't emit
  ~const~ qualifiers, so C code must drop ~const~ from ~Mode *~ etc.
  to match generated signatures for [[./39.org][#39]] swaps. Once [[./70.org][#70]] lands, the
  builder can emit ~const~ where appropriate and C can use it again.
