# priority: high
** OPEN #183 Adding ~///~ comments to core lib files forces full rebuild+test

Every ~///~ doc comment in a core library file (e.g. [[file:../../src/core/vec.til][src/core/vec.til]])
  gets embedded into ~boot/til.c~ during bootstrap, so the binary
  changes and a full ~make clean && make test~ cycle is required. For
  documentation-only changes this is unsustainable -- documenting a
  single function takes minutes of test suite time.

  Options:
  - Strip ~///~ comments during boot embed so they don't change the
    boot C output (doc registry is built at runtime from source, not
    from boot). Boot only needs the code, not the comments.
  - Cache the doc blob separately from the boot binary so doc-only
    changes don't invalidate the boot snapshot.
  - Add a ~make fast-test~ target that skips the full rebuild and only
    runs regression tests (the full suite is ~80% build tests that
    recompile examples irrelevant to a doc change).
  - In the merging procedure (~doc/bots/merging_from_remote.org~), skip
    the ~make test~ step when the only non-boot source change is
    ~///~ comments in core libs (risky: still need to ensure the
    parser embed didn't break).
