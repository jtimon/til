** OPEN #102 Auto-generate eq and to_str for structs

Structs only auto-generate ~clone~, ~delete~, ~size~. Enums get
  ~eq~, ~to_str~, ~clone~, ~delete~. But enum payloads require
  their type to have ~eq~ and ~to_str~, which forces manual stubs
  on every struct used as an enum payload.

  Auto-generate for structs (if not already defined):
  - ~eq~: field-by-field comparison (all non-namespace fields)
  - ~to_str~: struct name + field values

  This would eliminate the need for manual ~eq~/~to_str~ stubs on
  TypeBinding, StructInstance, EnumInstance, etc.
