** OPEN #274 Generated eq/to_str should support own/ref fields deeply

Issue [[../fixed/102.org][#102]] added generated ~eq~ and literal-style
  ~to_str~ for structs whose fields are all shallow and already support the
  needed method. It deliberately omits generation for structs with ~own~ or
  ~ref~ fields, because those need deeper ownership/lifetime decisions.

  Follow-up goal:
  - define whether generated ~eq~ compares owned/ref fields by value or by
    identity
  - define how generated ~to_str~ should print owned/ref fields without
    creating lifetime or constfolder/builder inconsistencies
  - update struct generation once those rules are clear

  This should be treated as a separate feature from shallow value structs;
  the interactions with constfolder and builder ownership are more complex.
