# priority: medium
** OPEN #193 Add lower-level File cursor APIs

Current file helpers mostly work in whole-file terms:

- read the full file into memory
- build a modified full string
- write the whole file back out

That is simple, but it is wasteful and awkward for tools that only need to
inspect or patch small regions.

Wanted direction:

- add lower-level File operations for cursor-based access
- expose read/write position movement explicitly
- support partial reads and writes without rewriting the whole file

Possible API shapes:

- seek / tell style methods
- fixed-size read/write methods on an open File
- cursor-relative helpers if that fits til better than POSIX naming

This should reduce memory churn and make small in-place file updates easier in
tools like the issue-summary generator and other source-processing utilities.
