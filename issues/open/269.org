** OPEN #269 Interface dynamic dispatch: polymorphic interface-typed values

   Follow-up to #251 / #268. Make an interface a real polymorphic type: a
   value of an interface type (`d: Drawable`) can hold any conforming concrete
   type, and a method call on it (`d.draw()`) dispatches at runtime to that
   concrete type's implementation. Today this is rejected at compile time
   ("argument type mismatch for 'd': expected Drawable, got Circle"); the goal
   is to make it work and dispatch correctly.

   Agreed approach (at least initially): reuse the existing ~Dynamic~ / dyn
   function-pointer machinery rather than building a separate vtable system.
   An interface-typed value is, under the hood, a boxed ~Dynamic~ carrying the
   concrete type tag plus the value. Passing a conforming concrete type where
   the interface is expected is a compiler-inserted coercion (box it) --
   allowed precisely because the #251 conformance check already proved the
   type implements the interface. A method call on an interface-typed value
   lowers to a dynamic dispatch (`dyn_call_ret`-style: look up the method for
   the runtime tag and call it). #268 is the prerequisite: because every
   implementer now has the full interface method set (provided abstracts +
   copied concrete defaults) physically in its namespace, the dyn lookup
   always finds a method to dispatch to, including the inherited defaults.

   Scope / design points:
   - Typer: accept a conforming concrete type where an interface type is
     expected (param, return, field, local, collection element); insert the
     box/coerce. Narrow the callable surface of an interface-typed value to
     exactly the methods the interface declares.
   - Lowering: emit the dynamic dispatch for `iface_val.method(...)` in both
     the interpreter and the C builder.
   - Memory/ownership: interface values go through Dynamic's existing
     clone/delete.
   - Performance: string-name dispatch is slower than a vtable index; fine to
     start, optimizable to real vtables / monomorphization later.

   Regression tests: a function `render(d: Drawable) returns Str` called with
   a Circle and a Polygon dispatching to each one's draw; an interface-typed
   value stored in a Vec and iterated; an inherited default (`d.describe()`)
   dispatching through the interface value. Both backends.
