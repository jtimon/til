** WIP #231 Use UFCS for standalone functions throughout the codebase

  til supports Uniform Function Call Syntax: ~x.func(args)~
  desugars to ~func(x, args)~. The typer's ~try_ufcs_rewrite~
  handles this automatically. We use UFCS aggressively for
  methods but almost never for standalone functions.

  Example: ~is_null(x)~ reads worse than ~x.is_null()~. The
  latter reads left-to-right as a pipeline.

  Target functions (standalone, first-param-is-receiver):

  | Function             | Prefix calls | UFCS form             |
  |----------------------+--------------+-----------------------|
  | is_null              |          547 | x.is_null()           |
  | expr_til_type        |          401 | e.expr_til_type()     |
  | ident_name           |          234 | id.ident_name()       |
  | set_expr_til_type    |          147 | e.set_expr_til_type(t)|
  | field_access_name    |           80 | fa.field_access_name()|
  | ptr_eq               |           13 | a.ptr_eq(b)           |
  | literal_num_text     |         few | n.literal_num_text()  |

  None of these currently use UFCS -- the ~is_null~ comment even
  says "Reached via UFCS as ~aaa.is_null()~" yet it is never
  called that way.
