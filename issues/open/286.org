# category: cleanups
** WIP #286 Interpreter: replace boxed Value with rstil's heap_index/EvalHeap raw-bytes model

  The interpreter's runtime value representation is the root cause of the
  recurring memory leaks (#275 and the long tail of follow-ups after it).
  Current master has the real-pointer ~EvalHeap~ and ~Scope.heap_index~
  groundwork, but eval still boxes every runtime result in ~Value~
  (~src/self/interpreter.til:331~). The heap-owning variants
  ~Value.Struct(StructInstance)~ and ~Value.Enum(EnumInstance)~ still carry
  per-value payload ownership, binding/passing still deep-clones the box,
  and teardown still runs hand-written helpers. The target is the rstil
  model with no ~Value~ box: a variable is NAME -> real heap pointer, the
  value is raw C-layout bytes, the type lives in scope metadata / the typed
  AST, and reclamation is one ~heap_drop(type, ptr)~ plus
  ~heap_free(ptr)~ per owning block. It is large; stage it.

*** Progress

  Done on current master:

  - ~src/self/eval_heap.til~ exists and uses real C heap pointers, not a
    growable offset buffer. This is the correct til port because payload
    bytes contain real pointers such as ~Str.c_str~, container ~data~, enum
    payload pointers, callable boxes, and FFI-visible addresses.
  - ~Context.eval_heap~, ~Scope.heap_index : Map(Str, Dynamic)~,
    ~scope_lookup_offset~, and ~scope_insert_offset~ exist. They are still
    mostly infrastructure: ordinary eval lookup/bind still uses
    ~Scope.bindings : Map(Str, Binding)~ -> ~Cell~ -> ~Value~.
  - ~heap_clone(struct_name, data, ctx)~ exists and can clone struct bytes by
    type/layout, including Str payloads and some container cases. It is still
    not the single canonical clone path: it calls old helpers and still
    manufactures / consumes boxed ~Value~ in enum and field paths.
  - ~StructInstance~ no longer carries ~struct_def~, but it still carries
    ~struct_name~, ~ctx~, ~is_str~, ~is_str_container~, ~own data~, and
    ~borrowed~. ~EnumInstance~ still carries ~enum_name~, payload data,
    size, borrowed, and Str-payload state. The boxed-payload model remains
    live.
  - The garbager is not the first blocker anymore. It already inserts
    scope/last-use deletes, assignment old-value deletes, own-transfer
    handling, clone-to-move rewrites for last-use aggregate assignments, and
    heap-promotion metadata. The remaining work is to make the interpreter
    execute those operations against raw heap pointers instead of ~Value~
    boxes.

  Remaining plan: staged source commits plus a final bootstrap-sensitive
  ~Two-pass:~ removal commit. Do not keep a thin scalar ~Value~ fallback; the
  end state removes ~Value~ completely.

*** Why the current model leaks (band-aids will not converge)

  ~Value.Struct(StructInstance { struct_name, ctx, data, borrowed, ... })~
  and ~Value.Enum(EnumInstance { enum_name, data, data_size, borrowed, ... })~
  are still a second memory model layered on top of raw C-layout buffers:

  - Per-value clone: ~Value.clone~, ~StructInstance.clone~, and
    ~EnumInstance.clone~ deep-copy boxes. The current ~heap_clone~ helper is
    progress, but its callers still wrap the result back into boxed
    ~Value~ / ~StructInstance~ ownership.
  - Per-value hand-free: ~struct_deep_free~, ~free_value~,
    ~free_value_full~, ~free_container_str_data~,
    ~free_container_str_data_cached~, ~clone_container_str_buffer~,
    ~free_temp_arg_value~, and ~free_non_own_temp_args~ are still present.
    Every one is a special case that must stay exactly symmetric with every
    clone, read, field-write, return, call, and FFI handoff path.
  - Type refs remain in values: ~StructInstance~ still needs ~struct_name~ and
    ~ctx~ so a value can free or clone itself. In the target model the value
    is only bytes; the scope/AST supplies the type at each operation.

  The garbager (~src/self/garbager.til~) already inserts asap-destruction
  ~delete~ calls into the AST -- it is NOT broken. The defect is that the
  interpreter evaluates those deletes against the ~Value~ boxes
  (struct_deep_free, which is structurally incomplete) instead of against
  raw heap bytes (heap_free, which is total). Same garbager, wrong runtime
  representation.

*** Target model

  - Runtime scope: name -> real heap pointer. The pointer is the address
    returned by ~EvalHeap.heap_alloc~, not a synthetic offset.
  - Type metadata: a binding's type comes from scope metadata mirrored from
    declarations/params/captures plus the typed AST at expression sites. It
    is not stored inside the value bytes.
  - Value bytes: scalars, structs, Str, enums, containers, Dynamic pointers,
    and callable boxes use the same in-memory layout that the C backend emits.
  - Clone/drop: one typed byte-level pair, ~heap_clone(type, ptr, ctx)~ and
    ~heap_drop(type, ptr, ctx)~, owns all deep-copy and deep-free logic. It
    handles Str c_str, container buffers/elements, enum payloads, own fields,
    funcptr/callable boxes, and nested aggregates.
  - Eval result: expression evaluation returns either a typed heap pointer /
    borrowed view or a direct control-flow result. It does not return a boxed
    ~Value~. A store decides whether to move, clone, borrow, or drop based on
    the typed AST and ownership metadata.

  Net target: no ~Value~, no ~Cell~, no ~StructInstance~, no ~EnumInstance~,
  no per-value clone/delete, no type refs inside values.

*** Bootstrap constraint (read FIRST)

  The ~Value~ enum's integer tags are baked into ~til_boot~'s committed C
  (that is why Sbyte/Ushort were appended at the end rather than inserted,
  interpreter.til:397-403). Any change that REMOVES or REORDERS Value
  variants must be staged across master merges with a preparation commit
  (see ~doc/self.org~ "When a preparation commit is needed" / "Staged
  migration across master merges"). Schedule all variant removals LAST,
  after the heap path is already live. The interpreter compiles itself, so
  every stage must also round-trip ~make two_pass~.

*** Staged migration plan

  Stage 1 -- Runtime heap-binding metadata.

    Keep ~Scope.bindings~ temporarily, but make ~Scope.heap_index~ the real
    owner for new migrated paths. Add the missing metadata beside it: binding
    type, ownership/ref state, moved-out state, and callable/closure metadata
    as needed. Add typed helpers for lookup, bind, overwrite, move, borrow,
    clone, drop, and scope teardown. The helpers must take explicit type
    metadata and real heap pointers; they must not inspect a ~Value~ to decide
    what they own.

  Stage 2 -- Canonical typed heap clone/drop.

    Finish ~heap_clone(type, ptr, ctx)~ and add its dual
    ~heap_drop(type, ptr, ctx)~ before changing storage. These replace the
    current scattered clone/free routines structurally, not by wrapping them.
    Cover primitive bytes, Str c_str, inline and own aggregate fields,
    containers and their elements, enum payloads (including Str payloads and
    nested enums), funcptr/callable boxes, Dynamic-owned pointers, refs/views,
    and closures. Existing helpers may delegate to the new pair during the
    transition, but the direction must be old -> new, never new -> old.

  Stage 3 -- Aggregates bind/read/write through the real heap.

    Migrate struct construction, Str construction, enum construction,
    container construction, field access, field assignment, payload aliases,
    switch/match enum reads, return propagation, and namespace aggregate
    fields so their durable storage is the heap pointer in ~heap_index~.
    Temporary expression results may still bridge to ~Value~ at unmigrated
    boundaries, but no scope-owned aggregate payload should live in
    ~Cell.val~ after this stage.

  Stage 4 -- Calls, refs, Dynamic, closures, and FFI use typed heap slots.

    Migrate parameter binding, capture binding, ref-mut primitive pointers,
    Dynamic autoboxing, callable boxes, FFI argument/result marshalling,
    CLI argv construction, and constfolder known/namespace values to typed
    heap slots. Delete the side ownership tables that only exist because
    ~Value.Dynamic~ and callable boxes hide raw pointers from scope teardown.

  Stage 5 -- Scalars move into heap bytes.

    Store Int/Byte/Sbyte/Short/Ushort/Int32/Uint32/Uint64/F32/Bool/None as
    typed heap bytes too. Conditions, numeric widening, comparisons, switch
    keys, primitive ext-function dispatch, and diagnostics read scalar bytes
    via typed helpers. This stage removes the last legitimate reason for
    expression eval to return ~Value~.

  Stage 6 -- Delete the boxed runtime.

    Remove ~Value~, ~Cell~, ~Binding.cell~, ~StructInstance~,
    ~EnumInstance~, ~make_struct_value~, ~make_enum_value~,
    ~borrow_value~, ~shallow_copy_value~, ~value_is_borrowed~,
    ~free_value~, ~free_value_full~, ~free_temp_arg_value~,
    ~free_non_own_temp_args~, ~struct_deep_free~,
    ~free_container_str_data~, ~free_container_str_data_cached~,
    ~clone_container_str_buffer~, and the remaining ~Value.clone~ call graph.
    This is the bootstrap-sensitive step because ~Value~ tags are baked into
    committed boot C. Make it a ~Two-pass:~ commit unless it needs a prep +
    removal pair across a master merge under ~doc/self.org~.

*** Deletions when done

  The whole boxed runtime disappears: ~Value~, ~Cell~ ownership, boxed
  Struct/Enum payloads, per-value borrowed/is_str/is_str_container/is_str
  flags, all boxed clone/free helpers, and the ownership side tables whose
  only job is to compensate for ~Value.Dynamic~ or callable boxes. The
  interpreter should shrink: the target is one typed heap clone/drop pair and
  direct byte reads/writes, not more special-case cleanup code.

*** Verification at every stage

  - Source stages: ~make clean && make test~.
  - Final ~Two-pass:~ removal stage: ~make clean && make test_two_pass~.
  - Keep ~test/eval_heap.til~ green in interpret and compiled paths.
  - Keep the strings leak probe from regressing and drive it toward zero true
    leaks:
      ~TIL_LEAK_PROBE=1 ASAN_OPTIONS=detect_leaks=1 bin/til_asan interpret test/strings.til~
      ~TIL_LEAK_PROBE=1 ASAN_OPTIONS=detect_leaks=1 bin/til_asan translate test/strings.til~
  - Add focused regression tests as each storage class migrates: struct field
    clone/drop, Str return/assign/reassign, enum payload clone/drop, container
    of Str/container of aggregate, ref payload alias writeback, Dynamic
    autobox/free, callable/closure field, FFI aggregate arg/result, and
    constfolder known values.

*** Notes

  - This SUBSUMES the #275 follow-up leak work: once values are raw bytes
    with one typed heap clone/drop pair, own-field / container / clone leaks
    disappear structurally rather than needing a new hand-free per case.
  - Do not add another boxed result type as a replacement for ~Value~. Short
    transition structs are acceptable only as typed pointer views and must not
    own payloads or clone/free themselves.
  - Existing garbager behavior should be reused, not reimplemented first. Add
    garbager work only where the raw-heap interpreter exposes a concrete
    missing clone/delete/move in the typed AST.
