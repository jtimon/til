** OPEN #195 Consider removing ~mode test~

  Now that ~til test foo.til~ works on any file regardless of mode and
  discovers test() decls inline (issue #51), the dedicated ~mode test~
  is largely syntactic sugar that flips ~run_tests=true~ on
  ~til interpret~ / ~til build~.  A user can write:

    mode lib
    test_foo := test() { ... }

  and run it with ~til test foo.til~ -- ~mode test~ no longer adds
  capability.  Decide whether to:

  - keep ~mode test~ as a convenience (running plain
    ~til interpret some_test_file.til~ will still run its tests), or
  - deprecate / remove it and migrate ~test/*.til~ + the makefile
    harness to use the host file's natural mode plus ~til test~.

  If we keep it, doc/self.org should explicitly note that mode test
  has no behavioral difference from ~lib~ + ~til test~.

  Possibility worth considering as part of this work: replace
  ~src/test_runner.til~ + ~src/tests.til~ + ~bin/tests~ with direct
  ~bin/til test test/X.til~ invocations from the Makefile.  The
  current harness wraps separate ~til interpret~ and ~til build~
  invocations because ~til test~ didn't exist; now it does, and
  collapses to one subcommand per file.  The harness also tracks
  per-test ms/mem and renders ~doc/totals.csv~ / ~img/totals.svg~
  though, so it's not a free swap -- those bits would need to move
  somewhere else (or be dropped if no longer needed).
