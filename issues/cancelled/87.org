** #87 ~priv~ keyword for private declarations

:DEPENDENCIES: [[../fixed/86.org][#86]]
  Duplicate of [[../fixed/179.org][#179]]. The ~priv~ keyword and the
  unused-priv check (the dependency on #86 mentioned here) both
  shipped under #179 and [[../fixed/187.org][#187]]. See those
  issues for the actual implementation. Cancelling this ticket as a
  rediscovery; #179 was opened without spotting the older #87.

  Original description (kept for the record):

  Everything in til is public by default. Add ~priv~ keyword to
  mark declarations as module-private (not visible to importers).

  #+begin_src til
  priv helper := func(x: I64) returns I64 { x.mul(2) }
  do_thing := func(x: I64) returns I64 { helper(x).add(1) }
  #+end_src

  Importing this module gives access to ~do_thing~ but not ~helper~.

  Depends on [[../fixed/86.org][#86]] (unused declarations): private
  functions that are only called internally would trigger unused
  errors without proper visibility tracking. The typer needs to know
  that ~priv~ functions are "used" if called within their own module.

  Like Rust's ~pub~ but inverted -- til defaults to public, you opt
  into private. Simpler for small programs (everything just works),
  explicit when you need encapsulation.
