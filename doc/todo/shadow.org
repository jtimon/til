* Plan: Fix Bug #65 - "Source 'X' not found in caller context"

** Overview
Rust/TIL divergence in interpreter causes 10 til_interpreted tests to fail.

** Root Cause: Variable Name Divergence

Rust (interpreter.rs:2099-2102) - shadows custom_type_name:
#+begin_src rust
let custom_type_name = &match custom_type_name.as_str() {
    "Dynamic" | "Type" => value_type_to_str(...),
    _ => custom_type_name.clone(),
};
// custom_type_name is now "I64" (shadowed)
#+end_src

TIL (interpreter.til:2142-2147) - uses different variable:
#+begin_src til
mut resolved_type_name := custom_type_name
if custom_type_name.eq("Dynamic") {
    resolved_type_name = value_type_to_str(...)
}
// custom_type_name still "Dynamic", resolved_type_name is "I64"
#+end_src

Then at line 2329, TIL uses custom_type_name instead of resolved_type_name.

** Fix Strategy

1. Both files use resolved_type_name (no shadowing)
2. Enable gcc shadow warnings to catch future cases
3. Local rule: forbid shadowing in Rust/C until TIL supports it
