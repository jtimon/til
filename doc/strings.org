* String Design in TIL

** Internal Representation

#+begin_src til
Str := struct {
    own mut data : U8 = 0   # pointer to buffer
    mut count := 0           # content length in bytes
    mut cap := 0             # allocated capacity, or sentinel
}
#+end_src

The ~cap~ field encodes ownership:

| cap          | meaning                                          |
|--------------+--------------------------------------------------|
| cap >= 0     | owned buffer, cap = allocated size               |
| CAP_LIT (-1) | points to static const C data (string literal)   |
| CAP_VIEW (-2)| borrowed view into another Str's buffer          |

For owned strings, ~count <= cap~. For literals and views, ~count~ is the
content length and ~cap~ is the sentinel.

Named constants ~CAP_LIT~ and ~CAP_VIEW~ are defined in str.til (globally
available via core import).

** String Creation Patterns

*** Literal Assignment
#+begin_src til
s := "hello"   # count=5, cap=CAP_LIT in codegen (points to static const)
#+end_src

In codegen, literal data lives in static const storage — no malloc for data.
Only the Str struct is heap-allocated. In interpreter, data is strndup'd
with cap=CAP_LIT (also not freed by til delete).

*** Owned (from operations)
#+begin_src til
s := a.concat(b)   # count = a.count + b.count, cap = count (owned)
s := a.clone()      # owned copy
#+end_src

*** View (from substr)
#+begin_src til
view := s.substr(2, 3)   # count=3, cap=CAP_VIEW, points into s.data
owned := view.clone()     # materializes view into owned string
#+end_src

Zero-copy but can dangle if source is freed or realloced. No borrow checker —
accepted risk. Call ~.clone()~ when you need ownership.

*** Pre-sized with Capacity
#+begin_src til
s := Str.with_capacity(100)   # count=0, cap=100
s.push_str("hello")           # count=5, cap=100 (in-place, no realloc)
#+end_src

** Deletion Semantics

#+begin_src til
delete := proc(own self: Str, call_free: Bool = true) {
    if self.cap.gte(0) {     # owned — free the buffer
        free(own self.data)
    }
    if call_free { free(own self) }
}
#+end_src

Negative cap (CAP_LIT or CAP_VIEW) means data is not owned — skip free.

** Clone

Clone always creates an owned copy regardless of source cap:

#+begin_src til
clone := func(val: Str) returns Str {
    own new_data : U8 = malloc(val.count)
    memcpy(new_data, val.data, val.count)
    return Str(data = new_data, count = val.count, cap = val.count)
}
#+end_src

** push_str on views/literals

~push_str~ panics when ~cap < 0~ (view or literal). Views and literals are
borrowed/static data — mutating them is not allowed. Call ~.clone()~ first
to get an owned copy, then push_str on that.
