# Fixed issues moved from issues.org.

* FIXED #4 FFI: call external C functions from til
  Users can call C functions by providing a .c file alongside their .til program.
  Declare signatures in til with ext_func/ext_proc, implement in C with til_
  prefix and pointer calling convention.

  Codegen path: ccodegen.c emits forward declarations, compile_c links user .c.
  Non-core ext_structs get typedefs and forward declarations.

  Interpreter path: dispatch.c compiles user .c to .so, dlopen/dlsym into
  FFIEntry map. libffi-based calling (no arity cap). ffi_cif precomputed per
  function. Flat struct representation (StructInstance.data is C-layout-compatible)
  passed directly as args and wrapped on return (including ext_struct types).

  CLI -l flag support: ctil parses -l flags from argv, forwards to both
  compile_c and ffi_init cc commands. Enables linking external libraries
  (e.g. -l raylib -l GL -l m).

  Companion .c file detection: ctil.c checks for program.c next to program.til.
  Limited but functional. Proper mechanism (command-line flag, pragma) deferred.

  Remaining limitation: til's pointer calling convention (til_I64 *, til_Str *)
  doesn't match natural C conventions (int, const char *). External libraries
  still require wrapper .c files. See #23 (shallow keyword) for the fix.

  Tests: src/examples/ffi/ext_module.til (both interpret and codegen).
  Demo: src/examples/raylib.til + raylib.c (raylib via FFI, both modes).

* FIXED #5 Pointers, Vec, Array, native Str: C-level memory and dynamic structures
  :DEPENDENCIES: #2 #3
  Evolved from 4 pointer primitives into a full inline-value memory model.
  Dynamic structs (Vec, Array) store values inline in raw buffers via memcpy.
  Str became a native til struct (was ext_struct with hidden fields and C
  methods). Auto-generated clone/delete/size work for all types, including
  structs with own fields pointing to raw buffers.

** Phase 1: Pointer primitives and Vec

*** FIXED Pointer primitives (ext_func)
    Added to core.til, implemented in ext.c, dispatch.c handles interpreter:

    malloc := ext_func(count: I64) returns Dynamic {}
    realloc := ext_func(own buf: Dynamic, count: I64) returns Dynamic {}
    ptr_add := ext_func(buf: Dynamic, offset: I64) returns ref Dynamic {}
    memcpy := ext_proc(dest: Dynamic, src: Dynamic, len: I64) {}
    free := ext_proc(own val: Dynamic) {}

    Inline value model: memcpy copies raw bytes into buffers, ptr_add returns
    a pointer into the buffer (pure arithmetic, no indirection). Values live
    directly in buffers — no shallow pointer copies.

    Interpreter: VAL_PTR holds raw void* (same as codegen). h_malloc returns
    calloc, h_ptr_add does (char*)buf + offset, h_memcpy calls C memcpy,
    h_free calls C free. val_to_ptr() helper extracts void* from any Value.

*** FIXED Dynamic as a real type
    ccodegen: TIL_TYPE_DYNAMIC → "void *". emit_as_ptr/emit_deref skip
    indirection for Dynamic. typer: numeric 0 accepted for Dynamic (null).

*** FIXED Parser: ref with explicit type annotation
    ref name : Type = expr (explicit type for Dynamic reinterpretation).

*** FIXED Type.size() auto-generation
    Computed from fields in initer.c: own/pointer fields = 8 bytes, inline
    primitives = their size, inline structs = recursive sum. Str.size() = 16
    (8-byte pointer + 8-byte I64). Vec.size() = 48 (6 fields × 8).

*** FIXED Vec and Array structs in core.til
    Vec: growable dynamic array. Array: fixed-size with typed element storage.
    Both use memcpy+free to move values inline into raw buffers.
    dyn_call1/dyn_call2/dyn_call1_ret/dyn_call2_ret for type-erased method
    dispatch (clone, delete) on buffer elements.

    Vec.push: memcpy(dest, val, elem_size); free(own val)
    Vec.get: returns ptr_add(data, i * elem_size) as ref Dynamic
    Vec.delete: dyn_call2(elem_type, "delete", ptr_add(...), false) per element
    Vec.clone: dyn_call1_ret(elem_type, "clone", ...) + memcpy per element

*** FIXED dyn_call dispatch (4 variants)
    dyn_call1, dyn_call2, dyn_call1_ret, dyn_call2_ret — ext_func wrappers
    for type-erased method calls. Interpreter builds synthetic AST call nodes.
    Codegen emits static dispatch functions with if-chains matching type name
    strings (memcmp on til_Str fields).

** Phase 2: Native Str (was ext_struct)

*** FIXED Str as native struct in core.til
    Replaced ext_struct (hidden fields, all C methods) with a regular struct:

    Str := struct {
        own data : U8 = 0
        cap := 0
        namespace:
        new, len, char_at, eq, concat, clone, delete, to_str,
        substr, contains, starts_with, ends_with
    }

    All methods implemented in til using ptr_add, memcpy, malloc, free.
    String literals remain special syntax that constructs Str instances.

*** FIXED I64.to_str and U8.to_str in til
    Replaced C ext_funcs with pure til implementations. I64.to_str builds
    digit string via repeated div/mod, allocates buffer, fills with memcpy.

*** FIXED Removed TIL_TYPE_STR from type system
    Str is now TIL_TYPE_STRUCT with struct_name="Str" everywhere:
    ast.h (removed enum value), typer.c, initer.c, precomp.c, ccodegen.c.

*** FIXED Interpreter: VAL_STR kept as runtime representation
    Str values use VAL_STR wrapping compiler-internal Str struct (same
    pointer+cap layout). Str() constructor intercepts struct instantiation
    to create VAL_STR from data (VAL_PTR/VAL_U8) + cap args. Field access
    on VAL_STR: .data → VAL_PTR, .cap → VAL_I64.

    VAL_PTR → VAL_STR reinterpretation at function call boundaries: when
    dyn_call passes ptr_add result to Str.delete/clone, the VAL_PTR is
    reinterpreted as VAL_STR based on param type annotation. Handles both
    ident args (borrowed cells) and expression args.

*** FIXED Codegen: til_Str typedef in ext.h
    til_Str { til_U8 *data; til_I64 cap } defined in ext.h so FFI code
    can use it. Generated C file skips emitting Str typedef (ext.h provides).
    String literals emit til_Str_lit("...", cap) helper. println/print/format
    emit static helpers with __attribute__((unused)).

*** FIXED Codegen: forward declaration ordering
    Split emit_struct_def into emit_struct_typedef + emit_struct_funcs.
    Generated C file order: all struct typedefs → string helpers → all
    function bodies (struct namespace + enum + top-level) → dyn_call
    dispatch → main. Prevents incomplete-type errors from cross-struct
    references.

*** FIXED Typer: own fields skip auto-clone in constructors
    Str(data = buf, cap = n) — the data field is own, so the typer must
    NOT insert clone (which would copy 1 byte instead of moving the whole
    buffer pointer). Constructor args for own fields are marked is_own_arg
    and use move semantics.

*** FIXED FFI updated for til_Str
    greet.c uses til_Str * (includes ext.h) instead of compiler-internal
    Str *. Ext_func forward declarations in generated code use
    type_name_to_c() for proper til_ prefixed types.

* FIXED #9 Tagged unions (enums): enum definitions with optional payloads
  til lacks enums and unions. Tagged unions (like Rust enums) combine both:
  variants are named alternatives, optionally carrying a payload. Essential
  for translating C code to til (C uses enums + unions extensively).

  Syntax follows struct conventions — EnumDef type annotation is optional
  (inferred from enum keyword). Trailing commas allowed.

    Color := enum { Red, Green, Blue }

    Token := enum {
        Num: I64,
        Name: Str,
        Eof,
    }

  Variants are accessed as namespace members: Color.Red, Color.Green.
  Payload variants are constructed as calls: Token.Num(42).
  Payload extraction via auto-generated is_Variant / get_Variant methods
  (no match statement yet).

  Enums have a namespace section just like structs, separated by
  namespace:. The initer auto-generates clone, delete, eq, and to_str
  if missing, same as structs.

** FIXED Phase 1: simple enums (no payloads)
   Variants are named integer constants (0, 1, 2, ...). Internally stored
   as I64. Namespace access for construction: Color.Red.

   Implemented:
   - AST: NODE_ENUM_DEF, TIL_TYPE_ENUM, TIL_TYPE_ENUM_DEF
   - Parser: parse_enum_def() — comma-separated variants, optional
     namespace: section (same separator as structs)
   - Initer: register enum defs, generate variant namespace fields (I64
     constants 0,1,2...), auto-generate eq/clone/delete (ext_func/ext_proc)
     and to_str (pure func with if-chain). Skips if user provides their own.
   - Typer: TIL_TYPE_ENUM alongside TIL_TYPE_STRUCT everywhere (13 sites),
     EnumDef type, enum variant type override in field access
   - Interpreter: enum ext method dispatch (eq/clone/delete), scope
     registration and init_ns for NODE_ENUM_DEF
   - ccodegen: emit_enum_def (typedef long long, variant value globals,
     eq/clone/delete C implementations), forward declarations, variant
     initialization in main
   - scavenger/precomp: NODE_ENUM_DEF alongside NODE_STRUCT_DEF
   - Test: src/test/enums.til (basic enum, eq, to_str, clone, func params,
     namespace section with custom method)

** FIXED Phase 2: payloads (tagged unions)
   Variants can carry typed payloads. Internally a tag + payload value.
   Simple enums (no payloads) keep their Phase 1 I64 representation.
   Payload extraction uses auto-generated per-variant is_/get_ methods.

   Detection: an enum has payloads if any variant has explicit_type set.
   Variant registry entries (non-namespace, nchildren==0) kept in body
   for downstream code to scan variant names, types, and tag indices.

   Implemented:
   - Parser: Variant: Type syntax in parse_enum_def — peek for TOK_COLON

   - Typer: skip variant registry entries (nchildren==0) in infer_body

   - Interpreter: enum dispatch branching on has_payloads — constructor
     dispatch (clone payload into VAL_ENUM), is_Variant (tag comparison),

     delete (free_value). Helper functions: enum_has_payloads,
     enum_variant_tag, enum_variant_type, clone_value, free_value,
     values_equal. interpreter_init_ns converts I64 tags to VAL_ENUM
     for no-payload variants in payload enums.
   - ccodegen: tag enum + struct with union for payload enums. Constructor

   - dispatch.c: VAL_ENUM case in free function

* FIXED #7 Precomp phase: compile-time macro evaluation and constant folding
  A precomp phase (between typer and scavenger) that evaluates macros and
  pure func calls at compile time, replacing them with literal results.
  Pipeline: lex → parse → type check → precomp → scavenge → interpret/codegen.

  Phase 1 — macro definition and evaluation:
  - typer.c: macros must declare a return type (like funcs). Macro bodies
    type-checked with in_func=0 (can call procs). Calls to macros from funcs
    allowed (is_proc=0 in initer).
  - precomp.h/c (new): post-typer pass that walks all bodies, finds macro
    calls, evaluates them via the interpreter, and replaces them with literal
    results. Tracks compile-time known values (literals, constants, hoisted
    temps) to validate macro arguments. Supports I64, U8, Str, Bool returns.
  - interpreter.h/c: exported scope_new, scope_set_owned, scope_free.
    Extracted interpreter_init_ns() helper for namespace field initialization
    (used by both interpret() and precomp).
  - ctil.c: calls precomp(ast, path) after type_check, before scavenge.

  Phase 2 — constant folding for pure funcs:
  - precomp.c: extended to also fold user-defined FUNC_FUNC calls when all
    args are compile-time known. Same mechanism as macros — interpreter
    evaluates, result replaces call with literal. Added func_names set
    alongside macro_names, generalized try_eval_macro → try_eval_call with
    require_known param (macros error on unknown args, funcs silently skip).
  - Scavenger naturally removes macro defs (all calls folded) and func defs
    where all call sites were folded away. No scavenger changes needed.
  - Tests: src/test/macros.til (macros: I64 arithmetic, constants, Str concat,
    proc calls in macro body, macro in func, Bool return; constant folding:
    basic func folding, nested folding, constant arg, Str-returning func).

* FIXED #6 Scavenger phase: remove unused declarations from the AST
  A post-typer pass that walks the AST and removes declarations not reachable
  from the program's entry points. CLI mode uses main as entry point; script
  mode uses all top-level executable statements.

  Two levels of trimming:
  1. Unused top-level declarations (structs, functions, procs) removed entirely
  2. Unused namespace methods within kept structs removed individually

  What was done:
  - scavenger.h/c (new): BFS reachability from entry points. Collects
    references via recursive AST walk: NODE_IDENT names, namespace calls
    (qualified "Type.method" names), namespace field access/assign, type
    annotations (param_types, return_type, explicit_type). For structs,
    only instance field defaults are walked when the struct is visited —
    namespace methods are walked individually via their qualified name.
    Filters program->children and struct body->children in place.
  - ctil.c: calls scavenge(ast, mode) after type_check, before
    interpret/codegen.
  - Example: println("hello") keeps only Str (delete), Bool (empty),
    println. Removes I64, U8, StructDef, FunctionDef, Dynamic, and all
    unused methods from kept types.

* FIXED #0 Complete basic memory management before extending pointer semantics
  Memory management is now fully automatic. The user never calls free or
  delete explicitly — ASAP destruction and typer-inserted clone/delete handle
  everything. No shallow copies, no leaks, in both interpreter and ccodegen.

  What was done:

** FIXED Eliminate shallow copies: typer-inserted clone calls
   The typer inserts Type.clone(x) at all 4 copy sites when the value is
   a NODE_IDENT:

   | Operation              | Status |
   | Declaration (y := x)   | FIXED  |
   | Assignment (y = x)     | FIXED  |
   | Field assign (o.f = x) | FIXED  |
   | Struct construction    | FIXED  |

   Works for all types — primitives via core.til clone, user structs via
   auto-generated clone. Interpreter uses move semantics (transfer ownership,
   null source cell) instead of deep copy. ccodegen uses pointer-assign for
   compound types, deref-assign for scalars.

   Struct constructors use field-by-field assignment (emit_ctor_fields)
   instead of compound literals. Inline compound fields use deref + free-wrapper
   to avoid sharing internal pointers with hoisted temps.

** FIXED Reassignment frees old value
   Typer inserts Type.delete(own var) before compound-type variable reassignment.
   Typer inserts Type.delete(obj.field, call_free) before field reassignment:
   call_free=true for own fields, call_free=false for inline compound fields.
   Interpreter also calls free_value before overwriting (handles scalars).
   ccodegen evaluates RHS before freeing old value to prevent reading freed
   memory when RHS references the same variable (e.g. counter = counter.add(1)).

** FIXED Custom clone/delete: user-defined override for auto-generated
   The initer checks for existing clone/delete in the struct namespace before
   auto-generating. If the user defines their own, the initer skips generation.

** FIXED Use-after-own error: catch transferred variables at compile time
   The typer detects use of a variable after ownership transfer via `own` and
   suggests the fix: own x.clone().

** FIXED ASAP edge cases
   - if/else branches: works correctly (last_use set to if statement index).
   - while loops: return triggers parent-scope cleanup, break/continue don't.
   - Closures: deferred (not yet supported).

* FIXED #3 own keyword for struct fields: heap-allocated owned pointers
  :DEPENDENCIES: #0
  Struct fields are normally inline (contiguous in parent allocation).
  An own field stores a heap pointer instead. Clone recursively clones it,
  delete recursively deletes it (with call_free=true).

  Syntax: own field := default, own mut field := default.

    Line := struct {
        own start := Point(x=0, y=0)
        own end := Point(x=0, y=0)
    }

  What was done:
  - ast.h: is_own on decl, is_own_field on Expr (field access/assign nodes)
  - parser.c: own keyword in struct body (merged with existing call-site own
    via lookahead to distinguish declaration vs expression)
  - initer.c: auto-generated delete passes call_free=true for own fields
  - typer.c: marks is_own_field on field access/assign. ASAP recognizes own
    field constructor args and own field assignments as ownership transfers
    (fcall_has_own_arg for struct constructors, expr_transfers_own for field
    assigns). Inserts Type.delete(obj.field, true) before own field
    reassignment to free the old value.
  - ccodegen.c: struct typedef emits pointer (til_Point *start), compound
    literal stores pointer directly (not deref), field access uses -> through
    own fields, emit_as_ptr skips & for own fields, field assign emits
    pointer assign.
  - Interpreter: no changes needed (all struct instances are heap Values).
  - Tests: src/test/own_fields.til (access, defaults, clone, nested assign,
    replace, clone independence).
