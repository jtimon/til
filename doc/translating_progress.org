** Known issues

*** Shallow Value passthrough in widen_numeric
C code does bitwise copy for unchanged Values. til's v.clone() deep-clones,
which breaks struct/enum ownership. 5 tests fail: tuples, dyn_maps, structs,
asting, kwargs. Options:
1. Add a shallow_copy/bitwise_copy primitive to til
2. Restructure callers so widen_numeric only receives numeric Values
3. Keep widen_numeric in C permanently

*** Raw pointer array construction in FFI type functions
build_struct_ffi_type needs to write ffi_type* pointers into a malloc'd
array at byte offsets. til rejects assigning ref FFIType to Dynamic slots
(type mismatch). Options:
1. Add store_ptr_at(buf: Dynamic, offset: U64, ptr: Dynamic) ext_func
2. Use memcpy(ptr_add(buf, offset), ptr, 8) -- ptr passed by ref gives address
3. Keep FFI type functions in C permanently

*** StructInstance.data byte offset access
str_view, make_str_value, make_str_value_own, read_field, write_field all
manipulate StructInstance.data with hardcoded sizeof offsets. These are
platform-dependent (sizeof(U8*) = 8 on 64-bit). Options:
1. Add ext_func helpers: str_data_ptr(inst), str_count(inst), etc.
2. Use Str struct constructor directly (needs the cached_str_def global)
3. Keep these functions in C and translate higher-level functions that call them
