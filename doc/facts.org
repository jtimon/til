* NOT_ANYMORE All arguments are passed by reference, both semantically and in both backends
  builder: shallow params are passed by value; non-shallow params are C pointers.
  interpreter: identifier args use scope_set_borrowed (shared Cell).
  No value_deep_copy — removed via move semantics.

* NOT_ANYMORE All values are heap-allocated in both backends
  builder: scalar locals (I64, U8, I16, I32, U32, U64, F32, Bool) are stack-allocated.
  Compound types (Str, structs, enums) and own/ref/global/mut-param-target locals remain heap-allocated.
  interpreter: all values still heap-allocated (internal Cell/Value representation).

* TRUE ext.h defines type aliases shared by both backends
  typedef I64 til_I64; typedef U8 til_U8; typedef Bool til_Bool.
  Used in ext.c, interpreter.c (directly), and builder output
  (via #include "ext.h").

* TRUE The typer rewrites the AST, not just type-checks it
  Desugars UFCS (instance.method) to namespace calls (Type.method), resolves
  named args to positional, fills default args, inserts ASAP delete calls,
  hoists fcall args into temporaries, and inserts Type.clone() at all 4 copy
  sites (decl, assign, field assign, struct constructor).
  Goal: backends can be simpler and more uniform.

* TRUE Struct fields are stored inline by default
  Non-own fields are contiguous in the parent struct's allocation. That's
  why field deletes pass call_free=false — only the top-level struct is
  freed. Own fields are pointers (separately heap-allocated).

* TRUE func and proc are distinct
  funcs are pure: must return a value, can't call procs (except panic).
  procs can have side effects and don't return values.

* TRUE ext_func and ext_proc declare builtins with C implementations
  Declared in core.til — either as direct namespace fields (e.g. I64.add)
  or at top-level (e.g. exit, free). Implementations live in ext.c with
  PascalCase names (til_I64_add, til_Bool_not, etc.). Interpreter dispatches
  via strcmp chain; builder skips them (ext.c provides the symbol directly).
  Direct namespace ext_funcs eliminated ~50 lines of trivial wrappers in core.til.

* NOT_ANYMORE Variadic parameters (..Type) only work in ext_func and ext_proc
  Fixed by #8: variadic params now work in regular funcs, procs, and macros.
  Variadic args are received as an Array inside the function body.

* TRUE Everything is immutable by default, mut opts in to mutation
  Applies to variable declarations (mut x := 5), struct fields (mut x := 0),
  and function/proc parameters (mut self: Point). The typer enforces this —
  assignments to non-mut bindings are type errors.

* TRUE The own keyword marks ownership transfer
  At call sites, own marks that the caller gives up ownership: own p.delete().
  On func/proc param defs, param_owns marks which params take ownership.
  ASAP destruction skips variables after an own transfer — the callee is
  now responsible for cleanup.

* TRUE Returns are owned by the caller unless declared ref
  Functions return heap-allocated pointers. The caller owns the returned
  value and ASAP handles cleanup. Functions can declare returns ref to
  return a borrowed reference — the caller must not delete it. ASAP
  skips cleanup for ref return values and ref declarations.
  Ref returns allow returning args without copying (e.g. min(a, b)).

* TRUE The only meta-types are StructDef, EnumDef, FunctionDef, and Dynamic
  Meta-types are type-system scaffolding, not data types. You never
  instantiate them as values. The initer skips clone/delete generation
  for these. Everything else (I64, U8, Bool, Str, user-defined structs
  and enums) gets auto-generated clone and delete.

* NOT_ANYMORE Primitive clone is defined in pure til in core.til
  Scalar clones (I64, U8, I16, I32, U32, U64, F32, Bool) are now ext_func
  returns shallow — identity functions in ext.c. Str.clone still uses
  format(val). The initer skips auto-generation when a clone already exists.

* TRUE The initer auto-generates clone and delete for structs
  clone: returns a new instance with each field cloned via FieldType.clone().
  delete: calls FieldType.delete(own field) for each field, then free(own self).
  Skipped if the user defines their own clone or delete in the struct namespace.
  Fieldless struct clone generates return Type() to avoid recursion.

* TRUE The typer inserts clone for the 3 ordinary copy sites
  When RHS/arg is NODE_IDENT, the typer wraps it in Type.clone(x).
  Covers: declarations (y := x), assignments (y = x), and field
  assignments (obj.f = x).

* TRUE There are no shallow copies in til — all copies are deep
  The typer inserts Type.clone() at the ordinary copy sites (decl,
  assign, field assign). Clone is always a deep copy — auto-generated
  clones recurse into fields, primitive clones produce new values.
  The interpreter uses move semantics (transfer ownership, null source cell)
  for identifier RHS — no copying without an explicit clone call.
  builder uses pointer-assign for compound types (clone returns fresh pointer),
  deref-assign for scalars. Struct literals are ownership-taking for all
  non-ref fields, so callers must use .clone() explicitly when they want
  to preserve a source variable.

* TRUE Memory management is fully automatic via ASAP destruction
  The typer inserts Type.delete() calls after the last use of each local
  variable, before interpretation or codegen. The user never calls free or
  delete explicitly. Works across all control flow: if/else branches,
  while loops, early returns, break, and continue. Auto-generated delete
  recurses into fields then frees the struct. Own-parameter transfer skips
  ASAP cleanup (callee is responsible). Mut variable reassignment frees the
  old value: typer inserts delete before compound-type reassignment,
  interpreter calls free_value for scalars.

* TRUE Every ext_func and ext_proc declaration exactly corresponds to its C header by qualifiers.org rules
  Verified 2026-03-10. Every parameter qualifier (shallow vs owned) matches the C
  signature (value vs pointer), every return qualifier matches (shallow → value type,
  non-shallow → pointer type), and every name matches after func_to_c renaming
  (sleep→sleep_ms, and→Bool_and, or→Bool_or, not→Bool_not, double→double_) and
  namespace flattening (Struct.method → Struct_method).

* TRUE The hoisting pass lifts fcall args into temporaries
  Prevents memory leaks from intermediate results and simplifies evaluation
  order. Runs inside the typer pass (in infer_body), before ASAP insertion.
