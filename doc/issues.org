* OPEN #8 Variadic args for funcs, procs, and macros
  Currently `..Type` variadic parameters only work in ext_func and ext_proc
  (format, println, print). Regular funcs, procs, and macros can't use them.

  What's needed:
  - Parser: allow `..Type` in func/proc/macro parameter lists
  - Typer: validate variadic calls, ensure args match the element type
  - Interpreter: collect variadic args into an array-like structure
  - ccodegen: emit variadic C calls or array-based passing

  This blocks several improvements:
  - Desugaring println/print in the typer (the replacement needs to be variadic)
  - User-defined variadic functions (e.g. custom logging, string builders)
  - Variadic macros for compile-time code generation

* OPEN #5 Pointers and arrays: unlock C-level memory access
  :DEPENDENCIES: #2 #3
  After #2 (ref) and #3 (own fields), til will have single-pointer semantics
  for structs, args, and returns — but still no way to work with multi-element
  allocations. C has malloc(sizeof(Elem)*n), pointer[i] indexing, pointer
  arithmetic, and dereference. Some of this is needed for self-hosting since
  the current C implementation uses these features (e.g. Expr.children, field
  arrays).

  own and ref will replace C's raw * for single pointers, giving the typer
  enough information for compile-time garbage collection (no explicit free).
  Arrays need the same treatment: the user gets power without the responsibility
  of manual free — ASAP and auto-generated delete handle cleanup.

  What's needed:
  - Array type: own arr: [I64] or similar — contiguous heap allocation of N elements
  - Indexed access: arr[i] or arr.get(i)
  - Length: arr.len()
  - Construction: [1, 2, 3] or Array(I64, size=10)
  - Ownership: arrays are owned by default, clone deep-copies, delete frees

  What's probably NOT needed (at least initially):
  - Raw pointer arithmetic (arrays with bounds checking are safer)
  - Manual dereference (* operator)
  - Void pointers

* OPEN #4 FFI: call external C functions from til
  The ext_func/ext_proc syntax already works for builtins. Extend it so users
  can call arbitrary C functions by providing a .c file alongside their .til
  program. The user declares signatures in til, implements them in C following
  the til pointer convention, and the compiler links them.

  First milestone — user-provided ext file:

    # square.c
    #include <stdlib.h>
    long long *til_square(long long *x) {
        long long *r = malloc(sizeof(long long));
        *r = (*x) * (*x);
        return r;
    }

    # program.til
    square := ext_func(x: I64) returns I64 {}
    assert_eq(square(5), 25)

  Later — mode-specific imports:
  A mode like "gui" could auto-link a library (e.g. raylib) and auto-import
  a .til file with ext_func/ext_proc signatures and wrapper structs, so the
  user just writes `mode gui` and gets access to the API.

* WIP #2 ref keyword: non-owning references for returns, declarations, and struct fields
  Currently all returns are owned by the caller (heap-allocated, caller must
  delete). This breaks when a function wants to return one of its args:

    min := func(a: I64, b: I64) returns I64 {
        if a.lt(b) { return a } else { return b }
    }

  Either a or b would be owned twice and deleted twice (double-free).

** FIXED ref returns
   returns ref I64 — the caller does NOT own the returned value and must
   not delete it. ASAP destruction skips ref return values.

    min := func(a: I64, b: I64) returns ref I64 {
        if a.lt(b) { return a } else { return b }
    }

   Typer validates: return value must be a param or ref variable.

** FIXED ref declarations
   Required when binding the result of a ref-returning function:
   ref my_min := min(34, 35). ASAP destruction skips ref variables.

   Typer validates: RHS must be a ref-returning function call or a
   ref/param variable. Cannot pass ref variable to 'own' parameter.

   ASAP lifetime extension: args to ref-returning calls have their
   lifetimes extended to at least the ref variable's last use, preventing
   use-after-free.

   Interpreter: ref from ident uses scope_set_borrowed (shared cell).
   ccodegen: ref decl emits pointer assignment (no malloc).

** ref struct fields
   A ref field stores a pointer to an existing value (not inline).
   clone does not deep-copy it, delete does not recurse into it —
   just copies/ignores the pointer.

   Open question: default initialization. Inline fields use literal defaults
   (x := 0), but ref fields point to external values that may not exist yet
   at construction time. Options include NULL/nil default, requiring the
   field at construction, or something else. TBD.

** args are already ref by default
   Function args are borrowed (ref) unless explicitly marked own.
   No change needed for args.

* OPEN #1 ccodegen/interpreter consistency: eliminate unnecessary special cases
  The ccodegen and interpreter should be as similar and minimal as possible.
  The typer/initer should do more AST rewriting so backends can be dumb and uniform.

  Architecture now has 3 layers:
  - ccore.h/c: raw C types (Str, I64, U8, Bool) + raw functions with natural
    C semantics (value params for scalars, raw int returns for comparisons)
  - ext.h/c: til_* typedefs + pointer-boxing wrappers for ccodegen's calling
    convention (all args/returns are heap pointers)
  - dispatch.h/c: interpreter's ext_function_dispatch, calls ccore directly

  ccodegen links ext.c + ccore.c and emits calls to til_* wrappers.
  The interpreter calls raw ccore functions via dispatch.c's strcmp chain.
  core.til uses direct ext_func namespace fields instead of func-wrapping-ext_func.
  println/print/format live in ext.c as variadic C functions. Struct constructors
  use compound literals instead of per-struct _new functions.

  All 4 clone sites in #0 are FIXED. value_deep_copy removed from interpreter
  via move semantics: NODE_IDENT RHS in decl/assign/field-assign/struct-constructor
  transfers ownership (copies value, nulls source cell). ASAP deletes of moved
  values are no-ops (val_none guards in dispatch.c + own-param guard in eval_call).
  Literal hoisting expanded to cover bare literals in return/assign/field-assign,
  removing ccodegen's literal return and literal assignment branches.

** FIXED ccodegen: free() special case
   Moved to ext.c as til_free(). ccodegen generic call path handles it.
   Interpreter also uses til_free() from ext.c.

** FIXED ccodegen: exit() special case
   Moved to ext.c as til_exit(). ccodegen generic call path handles it.
   Interpreter also uses til_exit() from ext.c.

** FIXED ccodegen: i64_to_str, u8_to_str, str_eq emitted inline
   Raw implementations in ccore.c. ccodegen links via ext.c wrappers.
   Interpreter calls ccore functions directly via dispatch.c.

** FIXED bool_to_str was a needless builtin
   Replaced with pure til: if b { return "true" } else { return "false" }.
   Removed ext_func, C wrapper from ccodegen, and strcmp case from interpreter.

** FIXED type aliases and PascalCase ext naming
   Added typedefs in ext.h: til_I64, til_U8, til_Bool, c_str. Used everywhere
   instead of raw C types. PascalCase function names (til_I64_add, til_U8_eq)
   match what ccodegen naturally emits for namespace methods.

** FIXED U8 stored as I64 in interpreter
   Interpreter Value union now has a .u8 field (til_U8 *). No more storing
   U8 values in .i64 with casts. All U8 handlers are clean one-liners.

** FIXED trivial ext_func wrappers in core.til
   Namespace methods that just wrapped an ext_func call (e.g. I64.add calling
   i64_add) are now direct ext_func declarations. No wrapper generated in C,
   call sites go straight to ext.c. Eliminates ~50 lines of boilerplate in
   core.til and removes the double-call overhead.

** FIXED println/print moved to ext.c
   Both backends had type-aware special cases (~45 lines in ccodegen, ~30 in
   interpreter) switching on I64/U8/Bool/Str. But println/print are declared
   ext_proc(..Str) — all args are already strings. Moved to ext.c as simple
   til_println(int n, ...) / til_print(int n, ...) variadic C functions.
   ccodegen emits til_println(n, args...) like format. Interpreter simplified
   to printf("%s", *arg.str) loop. Removed unwrap_to_str() optimization hack.

** FIXED format() moved to ext.c
   Was generated inline in ccodegen output (~10 lines of fprintf'd C code).
   Now lives in ext.c as til_format(int n, ...). ccodegen call emission unchanged
   (already emitted til_format(n, args...)). Interpreter handler unchanged
   (already Str-only). Generated C no longer needs <stdarg.h>.

** FIXED ccodegen: struct constructors (til_X_new functions)
   Replaced per-struct til_X_new() constructor functions with inline malloc +
   C99 compound literals. ccodegen now emits `*var = (til_Type){f1, f2, ...}`
   at statement level (NODE_DECL, NODE_RETURN, bare NODE_FCALL). Fieldless
   structs just malloc without assignment. Removed ~47 lines of _new function
   generation and forward declarations. Interpreter simplified to always use
   positional args (typer desugaring fills all fields with defaults).

** FIXED ccodegen: deref-copy in assignments causes use-after-free
   Was: hoisting turned clone fcalls into temps, ccodegen emitted *s = (*_t)
   (deref-copy sharing internal pointers), ASAP deleted _t → use-after-free.
   Fix: three changes working together:
   1. Typer inserts Type.delete(own var) before compound-type reassignment
      in insert_free_calls (frees old value before overwriting).
   2. Hoisting no longer lifts top-level RHS for NODE_ASSIGN — the clone
      fcall stays as RHS, no intermediate temp.
   3. ccodegen uses pointer-assign (var = expr) for NODE_FCALL/NODE_LITERAL_STR
      RHS instead of deref-copy. Scalars still use deref-assign (*var = val).
   Interpreter fix: ext_function_dispatch now nulls own-arg ident cells
   (ext dispatch evaluates by value, wasn't nulling cells like user-defined
   funcs do via borrowed cells). Also evaluates RHS before freeing old value
   to prevent reading freed memory when RHS references the same variable.
   NODE_FIELD_ASSIGN to inline compound fields: typer now inserts
   Type.delete(obj.field, call_free=false) before inline compound field
   reassignment. ccodegen emits &(obj->field) via emit_as_ptr, freeing
   only internals (not the inline slot). Interpreter uses free_value.

** FIXED interpreter: reassignment leaks old value
   Was: NODE_ASSIGN did cell->val = src->val without freeing old value.
   Fix: free_value(cell->val) before overwriting for scalars/Str/enum.
   For compound types (Str, struct, enum), the typer-inserted delete
   handles cleanup before the assignment reaches the interpreter.
   NODE_FIELD_ASSIGN also frees old inline field values.

** FIXED ccodegen: literal return values need heap allocation
   Hoisting now converts bare literals in return statements to temporaries.
   Return emission is uniform: either NODE_IDENT (return var) or struct
   constructor. Removed the literal-specific malloc+copy branch from ccodegen.

** ccodegen: declaration emission branches on RHS type
   Declaration emission branches: if RHS is NODE_FCALL or NODE_LITERAL_STR,
   assign pointer directly; otherwise malloc+copy. After literal hoisting,
   the NODE_LITERAL_STR branch was removed from NODE_ASSIGN. NODE_DECL still
   needs branching for hoisted temp decls (_t := 5) vs fcall decls (_t := foo()).
   Accepted — minimal remaining complexity.

** interpreter: dispatch.c still uses strcmp chain
   Dispatch extracted to dispatch.c — calls raw ccore functions (I64_add,
   Bool_not, etc.) instead of ext.c wrappers. But still a linear strcmp chain.
   Could be replaced with a function pointer table keyed by flat name.
   Remaining special cases that can't use the table: free (accesses scope).
   println, print, and format are variadic (special arg count handling).

** FIXED interpreter: free() accesses variable cell directly
   Added assertion that free() arg is NODE_IDENT. The handler accesses the
   variable's Cell via scope_get — the only builtin that reaches into the scope.
   Now guarded: crashes with clear error on non-ident args. Acceptable since
   free() is only called inside auto-generated delete bodies where the arg is
   always "self" (an ident).

** FIXED interpreter: value_deep_copy on NODE_IDENT
   Removed value_deep_copy. eval_expr NODE_IDENT now returns cell->val directly.
   Move semantics in NODE_DECL, NODE_ASSIGN, NODE_FIELD_ASSIGN, and struct
   constructor: when RHS is NODE_IDENT, copy value and null source cell.
   ASAP deletes of moved (val_none) values are no-ops via:
   - dispatch.c: val_none guards on all primitive _delete handlers
   - eval_call: skip call when first own param's arg is val_none
   Function args with NODE_IDENT use scope_set_borrowed (shared cell), matching
   ccodegen's pointer-passing semantics.

** FIXED namespace call path: no named arg resolution
   Ported full named-arg resolution (slot array, positional-after-named error,
   duplicate detection, default filling) from the regular fcall path into the
   namespace call path. User code like Point.delete(call_free=false) now works.
