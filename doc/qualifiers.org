* Qualifier-to-C Reference

Complete mapping of til qualifiers to generated C, by context.

** Arguments (function/proc parameters)

| til                       | C                | semantics                            |
|---------------------------+------------------+--------------------------------------|
| ~arg: Type~               | ~Type *arg~      | borrowed ref (read-only)             |
| ~mut arg: Type~           | ~Type *arg~      | mutable ref (callee can modify)      |
| ~own arg: Type~           | ~Type *arg~      | ownership transfer (callee frees)    |
| ~shallow arg: Type~       | ~Type arg~       | pass by value (copy on stack)        |

** Return types

| til                       | C return type    | semantics                            |
|---------------------------+------------------+--------------------------------------|
| ~returns Type~            | ~Type *~         | heap-allocated, caller owns          |
| ~returns shallow Type~    | ~Type~           | value return (on stack)              |
| ~returns ref Type~        | ~Type *~         | borrowed pointer, caller must NOT free |

** Locals

| til                       | C (scalar)       | C (compound)     | semantics                            |
|---------------------------+------------------+------------------+--------------------------------------|
| ~x := expr~               | ~Type x = ...~   | ~Type *x = ...~  | owned immutable local (default)      |
| ~mut x := expr~           | ~Type x = ...~   | ~Type *x = ...~  | owned mutable local                  |
| ~ref x : Type = expr~     | ~Type *x = ...~  | ~Type *x = ...~  | borrowed immutable local             |
| ~ref mut x : Type = expr~ | ~Type *x = ...~  | ~Type *x = ...~  | borrowed mutable local               |

Scalar types: I64, U8, I16, I32, U32, U64, F32, Bool.
Compound types: Str, user-defined structs, enums.

** Struct fields

| til                       | C struct member  | semantics                            |
|---------------------------+------------------+--------------------------------------|
| ~field := default~        | ~Type field~     | inline value                         |
| ~mut field := default~    | ~Type field~     | mutable inline value                 |
| ~own field := default~    | ~Type *field~    | owned heap pointer                   |
| ~own mut field := default~ | ~Type *field~   | mutable owned heap pointer           |

** Key rules

- ~type_name_to_c("I64")~ = ~I64 *~ (pointer form, for normal args/returns/locals)
- ~type_name_to_c_value("I64")~ = ~I64~ (value form, for ~shallow~ args/returns)
- In generated C, ~DEREF(x)~ dereferences a pointer: ~(*(x))~
- Args are borrowed by default, so explicit ~ref~ parameters are redundant
- Returns are owned by default, so explicit ~returns own Type~ is redundant
- Locals are owned by default, so explicit ~own x := ...~ / ~own mut x := ...~
  are redundant for normal locals
- Dynamic is the current exception: storing ~Dynamic~ in a local still
  requires explicit ~own~
- ~shallow~ is for args/returns only, not locals
- ~shallow~ params are already values — no ~DEREF~ needed, but need ~&(Type){val}~ to pass to pointer-expecting functions
- ~returns shallow~ functions return ~Type~ not ~Type *~ — callers must not dereference the result
- Scalar locals are stack values: ~I64 x = value~, not ~I64 *x = malloc(...)~
- Scalar locals use ~&(Type){x}~ when passed to pointer-expecting functions
- Delete calls for scalar locals are suppressed (no heap allocation to free)
- Locals are NOT hoisted if: ~own~, ~ref~, global, FuncPtr, or Dynamic
- TODO (#50): compound types should also be stack-allocated (~Type x~
  instead of ~Type *x = malloc~), passed as ~&x~ to functions
