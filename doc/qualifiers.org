* Qualifier-to-C Reference

Complete mapping of til qualifiers to generated C, by context.

** Arguments (function/proc parameters)

| til                       | C                | semantics                            |
|---------------------------+------------------+--------------------------------------|
| ~arg: Type~               | ~Type *arg~      | borrowed ref (read-only)             |
| ~mut arg: Type~           | ~Type *arg~      | mutable ref (callee can modify)      |
| ~own arg: Type~           | ~Type *arg~      | ownership transfer (callee frees)    |
| ~shallow arg: Type~       | ~Type arg~       | pass by value (copy on stack)        |

** Return types

| til                       | C return type    | semantics                            |
|---------------------------+------------------+--------------------------------------|
| ~returns Type~            | ~Type *~         | heap-allocated, caller owns          |
| ~returns shallow Type~    | ~Type~           | value return (on stack)              |
| ~returns ref Type~        | ~Type *~         | borrowed pointer, caller must NOT free |

** Locals

| til                       | C (scalar)       | C (compound)     | semantics                            |
|---------------------------+------------------+------------------+--------------------------------------|
| ~x := expr~               | ~Type x = ...~   | ~Type *x = ...~  | owned immutable local (default)      |
| ~mut x := expr~           | ~Type x = ...~   | ~Type *x = ...~  | owned mutable local                  |
| ~ref x : Type = expr~     | ~Type *x = ...~  | ~Type *x = ...~  | borrowed immutable local             |
| ~ref mut x : Type = expr~ | ~Type *x = ...~  | ~Type *x = ...~  | borrowed mutable local               |

Scalar types: I64, U8, I16, I32, U32, U64, F32, Bool.
Compound types: Str, user-defined structs, enums.

** Struct fields

| til                        | C struct member  | semantics                            |
|----------------------------+------------------+--------------------------------------|
| ~field := default~         | ~Type field~     | inline value (default)               |
| ~mut field := default~     | ~Type field~     | mutable inline value                 |
| ~own field := default~     | ~Type *field~    | owned heap pointer                   |
| ~own mut field := default~ | ~Type *field~    | mutable owned heap pointer           |
| ~ref field : Type = NULL~  | ~Type *field~    | borrowed pointer (caller owns data)  |
| ~ref mut field : Type = NULL~ | ~Type *field~ | mutable borrowed pointer             |

** Key rules

- ~type_name_to_c("I64")~ = ~I64 *~ (pointer form, for normal args/returns/locals)
- ~type_name_to_c_value("I64")~ = ~I64~ (value form, for ~shallow~ args/returns)
- In generated C, ~DEREF(x)~ dereferences a pointer: ~(*(x))~
- Args are borrowed by default, so explicit ~ref~ parameters are redundant
- Returns are owned by default, so explicit ~returns own Type~ is redundant
- Locals are owned by default, so explicit ~own~ on locals is redundant:
  ~own x := ...~, ~own mut x := ...~, ~own x : Type = ...~, and
  ~own mut x : Type = ...~ are all rejected
- ~Dynamic~ is the raw owned-pointer exception:
  - use ~buf : Dynamic = malloc(...)~ for owned allocations
  - use ~ref elem : T = buf~ or ~ref elem : T = ptr_add(buf, i)~ for typed access
  - a local owned ~Dynamic~ must either transfer ownership to an ~own~ arg/field
    or be explicitly freed with ~free(own buf)~
- ~shallow~ is for args/returns only, not locals or fields
- ~shallow~ params are already values — no ~DEREF~ needed, but need ~&(Type){val}~ to pass to pointer-expecting functions
- ~returns shallow~ functions return ~Type~ not ~Type *~ — callers must not dereference the result
- Scalar locals are stack values: ~I64 x = value~, not ~I64 *x = malloc(...)~
- Scalar locals use ~&(Type){x}~ when passed to pointer-expecting functions
- Delete calls for scalar locals are suppressed (no heap allocation to free)
- Ordinary non-~ref~, non-~Dynamic~ locals are stack-backed values and are passed
  as ~&x~ to pointer-expecting functions
- ~Dynamic~ locals stay on the raw-pointer path

** Internal representation

All qualifier contexts (args, returns, locals, fields) use the same
~OwnType~ enum: ~Own~, ~Ref~, ~Shallow~.

| Context | Default   | Meaning of Shallow       | Meaning of Own          | Meaning of Ref        |
|---------+-----------+--------------------------+-------------------------+-----------------------|
| Args    | ~Ref~     | pass by value (stack)    | ownership transfer      | (redundant, = default)|
| Returns | ~Own~     | value return (stack)     | (redundant, = default)  | borrowed pointer      |
| Locals  | ~Shallow~ | owned stack value        | (forbidden, = default)  | borrowed pointer      |
| Fields  | ~Shallow~ | inline value in struct   | owned heap pointer      | borrowed pointer      |

For ~Param~ and ~FunctionDef~, the enum is used directly (~own_type~,
~return_own_type~).  For ~Declaration~ (locals and fields), the same
~own_type~ field replaces the former ~is_own~ / ~is_ref~ booleans.
~Shallow~ is the natural default for declarations: inline for fields,
stack-owned for locals.


** Visibility (priv keyword)

   Issue #179. til is public by default; the ~priv~ keyword opts
   individual declarations out of visibility. This is the inverse
   of Rust, where items are private by default and ~pub~ opts them
   in to exposure. The default keeps libraries that expose almost
   everything noise-free; the unusual case (genuinely internal
   helpers) carries the marker.

*** Where ~priv~ may appear

    | Context                                  | Effect                                         |
    |------------------------------------------+------------------------------------------------|
    | Top-level ~func~ / ~proc~ declaration    | hidden from importers (cross-file only)        |
    | Top-level ~struct~ declaration           | type name + constructor hidden from importers  |
    | Top-level ~enum~ declaration             | type name + variants hidden from importers     |
    | Top-level value (~mut~ / immutable)      | name hidden from importers                     |
    | Field inside a ~struct~ body             | hidden outside the struct's namespace methods  |
    | Function inside a ~namespace:~ block     | hidden outside the struct's namespace methods  |

*** Cross-file vs in-struct rules

    Top-level ~priv~ items follow a *cross-file* rule: they are
    visible everywhere within the declaring file (top-level helpers,
    other top-level functions, even outside the struct that uses
    them) and invisible from any importer.

    Struct members (fields and namespace methods) marked ~priv~
    follow a stricter *in-struct* rule: they are visible only from
    inside the struct's own namespace methods. Same-file code that
    is not part of the struct cannot touch them either.

*** Examples

    #+begin_src til
    // Library file: lib_helper.til
    mode lib

    // Public function -- visible to importers.
    add_one := func(x: I64) returns I64 {
        return x.add(internal_inc(1))
    }

    // Private to this file -- importers cannot reference it.
    priv internal_inc := func(x: I64) returns I64 {
        return x.add(1)
    }

    // Private struct -- importers cannot construct it or name it.
    priv Secret := struct {
        mut value := 0
    namespace:
        show := func(self: Secret) returns I64 { return self.value }
    }

    // Public struct with a private field and a private method.
    Box := struct {
        mut tag := 0
        priv mut hidden := 0
    namespace:
        // Public method may freely read the priv field of the
        // same struct.
        hidden_plus_one := func(self: Box) returns I64 {
            return self.hidden.add(1)
        }
        // Private namespace method -- only callable from another
        // Box method.
        priv bump := func(mut self: Box) {
            self.hidden = self.hidden.add(1)
        }
        // Public wrapper that calls the priv method internally.
        bump_and_show := func(mut self: Box) returns I64 {
            Box.bump(self)
            return self.hidden
        }
    }

    // Private top-level globals -- importers cannot read or write.
    priv MAGIC : I64 = 42
    priv mut counter := 0
    #+end_src

*** Not in scope

    - No module/crate-level visibility levels (no ~pub(crate)~,
      no ~pub(super)~). til's import graph is flat: a thing is
      either visible to importers or it isn't.
    - ~priv~ has no effect on builtins, ~ext_func~, ~ext_proc~,
      or ~ext_struct~.
    - The keyword is purely additive. Any existing declaration
      stays public unless ~priv~ is added explicitly.

*** Diagnostics

    - Cross-file violation: ~error: 'X' is private to '<decl-file>'~
    - In-struct violation:  ~error: 'Type.member' is private to its struct~
